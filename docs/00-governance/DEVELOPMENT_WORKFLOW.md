# 标准开发流程

业务需求分析
→ 业务规则确认
→ 功能范围规划
→ 业务流程详细设计
→ 数据库设计
→ 权限设计
→ 页面与交互设计
→ API 设计
→ PRD 确认
→ Codex 开发
→ 自动测试
→ 人工测试
→ 验收
→ 文档回写
→ Git 提交
→ 进入下一阶段

## 强制规则

- 每个阶段完成后必须更新文档；
- 每个阶段必须经过项目负责人确认；
- 未确认不得进入下一阶段；
- 本次任务结束后不得进入数据库设计。
