# 变更控制规则

提出变更
→ 确认变更原因
→ 识别受影响业务规则
→ 分析数据库、接口、页面和历史数据影响
→ 项目负责人批准或拒绝
→ 更新相关文档
→ 更新 `DECISION_LOG`
→ 更新 `CHANGELOG`
→ Codex 开发
→ 回归测试
→ 验收

Frozen 规则不得在没有正式变更记录的情况下修改。
