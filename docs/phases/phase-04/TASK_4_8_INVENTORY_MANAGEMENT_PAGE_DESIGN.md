# Task 4.8：库存管理页面设计（Inventory Management Page Design）

## 1. 文档状态与设计依据

- Task：Task 4.8 库存管理页面设计
- 状态：Completed / Approved
- 当前 Phase：Phase 4 页面设计（UI / Page Design，In Progress）
- 下一任务：Task 4.9 出入库管理页面设计
- Task 4.9 状态：Not Started
- 技术开发：Not Started

本任务严格依据 Task 4.1 至 Task 4.7、Frozen 业务规则、Task 2.5 状态流转设计及 Database Logical Design v1.0，只定义库存管理页面结构、信息分组、组件、状态、权限和跳转关系。

本任务不新增库存字段、库存表、库存对象或平行数据源，不定义 API、ORM、DDL、Schema、Migration、Seed，不编写页面代码，不安装依赖，也不创建 Task 4.9 至 Task 4.10 正文。

## 2. 核心 Frozen 映射规则

### 2.1 库存余额唯一数据源

正式库存余额必须严格映射 `inventories`，其唯一粒度为 SKU 与仓库。

正式数量及时间字段为：

- `on_hand_quantity`：账面现存数量；
- `available_quantity`：可用数量；
- `reserved_quantity`：预留数量；
- `pending_quantity`：待处理数量；
- `last_transaction_at`：最后流水时间；
- `last_counted_at`：最后盘点时间。

不得新增独立在途数量、独立锁定库存、待入库库存或其他 Frozen 中不存在的库存余额字段；不得建立独立厂家库存表或海外库存余额表。

### 2.2 在途库存表达

Frozen 数据库不设置 `in_transit_quantity`。在途库存统一由在途类型仓库及 `inventories` 中对应在途仓节点库存表达。

不得在同一库存记录中额外计算并保存独立在途数量，也不得同时叠加在途仓库存与独立在途字段造成重复统计。

### 2.3 仓库类型库存视图

公司仓、厂家仓、在途仓、海外仓、待处理仓及其他已批准仓库类型库存，均为 `warehouses` 与 `inventories` 统一数据的受控筛选或汇总视图。

这些视图不建立平行库存对象，不改变库存粒度，也不形成第二套正式余额来源。

### 2.4 库存流水

库存流水严格映射 `inventory_transactions`：

- 只追加；
- 不得编辑；
- 不得删除；
- 必须关联来源单据及来源明细；
- 不得通过库存页面手工创建无来源流水。

### 2.5 调拨、盘点和库存调整边界

库存管理可以查询调拨、盘点和库存调整结果并提供跳转入口，但相关业务的创建和执行页面归属 Task 4.9 出入库管理页面设计。本任务不提前设计这些业务的创建或执行正文。

### 2.6 库存预警

库存预警严格映射 `inventory_alerts`，只使用 Frozen 已批准的预警类型、阈值数量、触发时当前数量、预警级别、预警状态、查看信息、处理信息和关闭信息。

预警状态严格使用新预警、已查看、处理中、已解决、已忽略和已关闭，不新增预警类型、字段、状态或计算规则。

## 3. 页面定位

库存管理用于：

- 查询当前库存余额；
- 查看不同仓库类型库存；
- 查看 SKU 分仓库存；
- 查看库存流水；
- 查看和处理库存预警；
- 查看最后库存变动与最后盘点时间；
- 跳转相关来源业务单据。

库存管理不负责采购验收、生产验收、实际入库、实际出库、调拨执行、盘点执行、库存调整执行或直接修改库存余额。

当前库存页面由以下区域组成：

1. 页面标题与面包屑；

3. 操作工具栏；
4. 查询筛选区；
5. 当前库存表格；

- 最后盘点日期范围。

库存状态只能根据 Frozen 数量字段实时计算并用于筛选，不新增持久化库存状态字段。

### 4.3 表格字段

表格主要展示：

- SKU 编码；
- SKU 名称；
- 规格摘要；
- 仓库编码；
- 仓库名称；
- 仓库类型；
- 账面现存数量；
- 可用数量；
- 预留数量；
- 待处理数量；
- 最后流水时间；
- 最后盘点时间；
- 预警状态；
- 操作。

### 4.4 正式规则

- 数量字段直接映射 Frozen 数据；
- 不显示 Frozen 中不存在的待入库数量或独立在途数量字段；
- 在途仓记录通过仓库类型识别；
- 无权限仓库数据不得显示或参与汇总；
- 页面不得直接编辑库存数量；
- 当前库存记录不得通过页面物理删除。

## 5. 库存概况

顶部概况可展示：

- SKU 库存记录数；
- 账面现存总量；
- 可用总量；
- 预留总量；
- 待处理总量；
- 预警记录数。

可按公司仓、厂家仓、在途仓、海外仓、待处理仓及其他已批准仓库类型汇总。

正式规则：

- 汇总只使用当前用户有权限访问的库存记录；
- 在途库存只统计在途类型仓库；
- 不得重复叠加跨境发货数量和在途仓库存；
- 不得将采购待到货、生产待完工、待验收或待入库数量计入正式库存；
- 概况数据只是 `inventories` 的实时汇总，不形成独立数据源。

筛选条件包括生产厂家、厂家仓、SKU、基于正式数量计算的库存状态及预警状态。

表格展示：

- 生产厂家；
- 厂家仓；
- SKU；
- 账面现存数量；

- 页面不得直接执行跨境发货或修改库存。

## 14. 页面状态

库存管理页面必须覆盖：

- 正常状态；
- 首次加载；
- 局部加载；
- 无库存记录；
- 零库存；
- 无库存流水；
- 无预警；
- 无权限；
- 加载失败；
- 库存数据发生变化；
- 并发刷新；
- 预警已被其他用户处理；
- 来源单据无法访问。

正式规则：

- 库存变化时重新读取最新数据，不允许静默覆盖；
- 页面缓存不得成为正式库存来源；
- 局部加载失败不得阻断其他库存区域；
- 无权限时不得泄露仓库、数量、成本或金额信息；
- 来源单据无权限或不存在时显示明确状态，不暴露敏感信息。

- 当前库存查看；
- 分仓库存查看；
- 厂家仓库存查看；
- 在途库存查看；
- 海外仓库存查看；
- 库存流水查看；
- 库存成本查看；
- 库存金额查看；
- 库存预警查看；
- 库存预警处理；
- 库存预警关闭；
- 库存导出。

正式规则：

- 页面数据必须受仓库数据权限控制；
- 跨境验收人员只查看授权厂家仓；
- 无成本或金额权限时隐藏对应字段；
- 前端隐藏不替代服务端权限校验；
- 本任务不新增角色、权限表或权限数据结构。

## 16. 页面跳转关系

库存记录可跳转 SKU 详情、仓库详情、库存流水、库存预警，以及来源入库单、出库单、调拨单、盘点单、库存调整单和跨境发货单。

库存流水可跳转来源单据、来源单据明细、关联流水和 SKU 库存详情。

库存预警可跳转 SKU 库存详情、仓库库存详情，以及相关采购、生产、出入库或跨境业务页面。

所有跳转必须遵循 Tab 复用、功能权限和仓库数据权限。跨模块入口只确定跳转方向，不提前设计 Task 4.9 或 Task 4.10 正文。
