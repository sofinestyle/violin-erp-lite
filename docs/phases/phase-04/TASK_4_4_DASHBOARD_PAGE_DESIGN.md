# Task 4.4：首页 Dashboard 页面设计（Dashboard Page Design）

## 1. 文档状态

- Task：Task 4.4 首页 Dashboard 页面设计
- 状态：Completed / Approved
- 当前 Phase：Phase 4 页面设计（UI / Page Design，In Progress）
- 下一任务：Task 4.5 基础资料模块页面设计
- Task 4.5 状态：Not Started
- 技术开发：Not Started

## 2. 文档目的

本文档在已批准的 Task 4.1 页面架构设计、Task 4.2 PC 管理端布局与导航设计和 Task 4.3 PC 管理端视觉规范设计基础上，确定 PC 管理端首页 Dashboard 的页面定位、区域结构、页面组件、跳转关系、权限表现及页面状态，为后续页面设计提供正式依据。

## 3. 任务范围

本任务仅确定：

- PC 管理端首页 Dashboard 页面定位；
- 页面区域划分；
- 待我处理区域；
- KPI 指标卡片；
- 趋势和库存结构图表；
- 最近业务动态；
- 快捷入口；
- 页面组件；
- 页面跳转；
- 页面权限；
- 加载、空状态和异常状态。

本任务不包含 Task 4.5 至 Task 4.10 正文、具体业务字段技术实现、图表库或前端组件库选型、页面代码、API、数据库、ORM、DDL、Schema、Migration、Seed、技术框架、Figma、可运行原型或依赖安装。

## 4. 设计依据

Dashboard 必须遵循：

- Task 4.1 页面架构设计；
- Task 4.2 PC 管理端布局与导航设计；
- Task 4.3 PC 管理端视觉规范设计；
- Frozen 业务规则；
- Database Logical Design v1.0。

固定要求：

- 使用 Header、Sidebar、Tab、Main Content 布局；
- 首页 Tab 固定且不可关闭；
- Info Panel 第一版默认折叠；
- 第一版仅支持 Light Mode；
- 页面文字使用中文（简体）；
- Dashboard 不直接修改任何业务数据和库存数据。

## 5. 页面定位

Dashboard 是用户登录后的默认首页。

### 5.1 核心职责

- 展示当前用户可见的业务概况；
- 聚合当前用户待办事项；
- 展示库存和业务预警；
- 展示经营和库存关键指标；
- 提供高频业务快捷入口；
- 提供最近业务动态入口。

### 5.2 不承担的职责

- 完整统计报表；
- 财务报表；
- 复杂经营分析；
- 业务单据编辑；
- 库存调整；
- 审核操作本身。

用户点击 Dashboard 内容后，进入对应业务页面完成处理。

## 6. 页面总体结构

页面从上至下分为：

1. 页面头部；
2. 待我处理；
3. KPI 指标卡片；
4. 业务趋势与库存结构；
5. 最近业务动态；
6. 快捷入口。

结构示意：

```text
页面头部
↓
待我处理
↓
KPI 指标卡片
↓
采购趋势｜库存结构｜仓库库存占比
↓
最近业务动态｜快捷入口
```

页面整体使用 Task 4.3 确定的白色卡片、浅灰背景和企业蓝主色。

## 7. 页面头部

页面头部显示：

- 页面标题：首页；
- 当前用户问候信息；
- 当前日期；
- 当前组织或公司；
- 数据统计时间范围；
- 刷新按钮；
- 最近更新时间。

时间范围可提供：

- 今日；
- 本月；
- 最近 30 天。

正式规则：

- 时间范围只影响 Dashboard 展示数据；
- 不改变其他业务页面筛选条件；
- 刷新操作只重新获取展示数据；
- 刷新不得修改业务数据；
- 数据加载中应显示局部加载状态；
- 刷新失败时应保留原有已加载数据并提示失败原因。

## 8. 待我处理（My Tasks）

待我处理是 Dashboard 的最高优先级区域。

第一版可聚合：

- 待审核采购订单；
- 待采购验收；
- 待生产验收；
- 待入库；
- 待出库；
- 待处理库存预警；
- 即将逾期采购订单；
- 即将逾期生产单；
- 待确认跨境发货；
- 待确认海外收货。

每项显示：

- 待办名称；
- 待办数量；
- 最高优先级；
- 最早到期或逾期时间；
- 查看全部入口。

正式规则：

- 只显示当前用户有权限处理或查看的事项；
- 无权限事项不得显示；
- 数量为 0 时可隐藏或显示 0，具体统一规则留后续实现阶段确定；
- 逾期事项使用警告或错误状态；
- 点击待办后进入对应业务列表；
- 进入列表时自动带入对应筛选条件；
- Dashboard 不直接执行审核、验收、入库或出库操作。

建议跳转关系：

| 待办 | 跳转方向 |
| --- | --- |
| 待审核采购订单 | 采购管理 / 采购订单 / 待审核 |
| 待采购验收 | 采购管理 / 采购验收 / 待验收 |
| 待生产验收 | 生产管理 / 生产验收 / 待验收 |
| 待入库 | 出入库管理 / 入库管理 / 待入库 |
| 待出库 | 出入库管理 / 对应出库列表 |
| 待处理库存预警 | 库存管理 / 库存预警 |
| 即将逾期采购订单 | 采购管理 / 采购执行跟踪 |
| 即将逾期生产单 | 生产管理 / 生产进度 |
| 待确认跨境发货 | 跨境业务 / 跨境发货 |
| 待确认海外收货 | 跨境业务 / 海外收货 |

## 9. KPI 指标卡片

第一版建议展示八项指标：

1. 当前 SKU 数；
2. 当前库存总数量；
3. 公司仓库存；
4. 厂家仓库存；
5. 在途库存；
6. 海外库存；
7. 本月采购金额；
8. 本月入库数量。

每张 KPI 卡片包含：

- 指标名称；
- 当前数值；
- 数据统计时间；
- 简单变化提示；
- 对应详情入口；
- 必要的单位。

展示规则：

- 数量使用千分位；
- 金额格式示例：`¥12,345.00`；
- 海外库存必须显示数据更新时间；
- 变化提示仅用于辅助，不作为财务结论；
- 无权限指标不显示；
- 无数据时显示 0 或“暂无数据”，不得显示错误占位值；
- 不使用大量不同颜色卡片；
- 主要指标使用企业蓝和中性色；
- 异常指标才使用警告或错误状态色。

建议跳转关系：

| KPI | 跳转方向 |
| --- | --- |
| 当前 SKU 数 | 基础资料 / SKU 管理 |
| 当前库存总数量 | 库存管理 / 当前库存 |
| 公司仓库存 | 库存管理 / 分仓库存 |
| 厂家仓库存 | 库存管理 / 厂家库存 |
| 在途库存 | 库存管理 / 在途库存 |
| 海外库存 | 库存管理 / 海外库存 |
| 本月采购金额 | 统计分析 / 采购统计 |
| 本月入库数量 | 统计分析 / 出入库统计 |

Task 4.4 仅确定跳转方向，不开始统计分析页面正文设计。

## 10. 图表区域

第一版设置三个图表。

### 10.1 最近 30 天采购趋势

展示：

- 日期；
- 采购订单金额；
- 采购订单数量。

建议使用折线图或柱线组合图。

正式规则：

- 可切换最近 7 天、最近 30 天和本月；
- 图表点击进入采购统计或采购订单列表；
- 图表仅展示当前用户有权限查看的数据；
- 无数据时显示明确空状态。

### 10.2 库存结构

展示仓库类型：

- 公司仓；
- 厂家仓；
- 在途仓；
- 海外仓；
- 待处理仓。

建议使用环形图。

正式规则：

- 同时显示数量和占比；
- 点击某一类型进入对应库存列表；
- 不得将待入库数量计入正式库存，除非 Frozen 规则明确支持；
- 海外库存必须显示数据日期。

### 10.3 仓库库存占比

按仓库展示：

- 仓库名称；
- 库存数量；
- 库存占比；
- 预警 SKU 数量。

建议使用横向条形图。

正式规则：

- 默认展示库存数量前若干仓库；
- 支持查看全部；
- 点击仓库进入分仓库存；
- 用户只可看到授权仓库。

本任务只确定图表信息与建议表现，不选择具体图表库。

## 11. 最近业务动态

最近业务动态显示当前用户数据权限范围内的最近业务事件。

事件类型可包括：

- 新增采购订单；
- 采购订单审核；
- 委外生产单创建；
- 生产进度更新；
- 分批完工；
- 采购验收；
- 生产验收；
- 入库；
- 出库；
- 调拨；
- 跨境发货；
- 海外收货；
- 海外库存导入。

每条动态显示：

- 业务类型；
- 单据编号；
- 简要描述；
- 操作人；
- 操作时间；
- 业务结果或状态；
- 查看详情入口。

正式规则：

- 按时间倒序排列；
- 默认显示最近若干条；
- 提供查看全部入口；
- 只显示用户有权访问的数据；
- 点击后进入对应详情页；
- 不在 Dashboard 中直接编辑业务记录。

## 12. 快捷入口

第一版建议提供：

- 新建采购订单；
- 新建委外生产单；
- 采购验收；
- 生产验收；
- 新建入库单；
- 新建出库单；
- 当前库存；
- 新建跨境发货。

正式规则：

- 根据功能权限动态显示；
- 无新增或执行权限时不显示；
- 跨境验收人员只显示授权的生产验收入口；
- 仓库人员只显示授权仓库相关入口；
- 点击后进入对应独立页面或列表；
- 快捷入口不得绕过审核、验收或库存规则。

| 页面反馈 | 骨架屏、局部加载、空状态、错误提示、无权限状态 |

## 14. 页面权限

Dashboard 显示内容必须同时考虑：

- 用户角色；
- 功能权限；
- 仓库数据权限；
- 店铺数据权限；
- 厂家及厂家仓数据权限。

正式规则：

- 无权限模块不显示；
- 无查看权限的数据不得参与 KPI 和图表汇总；
- 公司负责人可查看授权范围内的经营概况；
- 采购人员主要看到采购待办及采购数据；
- 仓库人员主要看到入库、出库、盘点和库存预警；
- 跨境验收人员主要看到授权厂家相关生产验收；
- 销售人员只显示授权店铺或销售相关数据；
- 权限控制不得只依靠前端隐藏；
- Task 4.4 只定义页面表现，不定义技术实现。

## 15. 页面状态

### 15.1 正常状态

所有有权限模块正常展示。

### 15.2 首次加载

使用骨架屏或局部加载状态。

### 15.3 局部加载

某个卡片或图表单独加载，不阻塞整个页面。

### 15.4 部分失败

- 某个模块加载失败时，其余模块继续展示；
- 失败模块显示重试入口；
- 不清空其他已成功数据。

### 15.5 无业务数据

显示明确空状态和可用快捷入口。

### 15.6 无权限

如果用户没有任何 Dashboard 数据权限：

- 显示简洁无权限说明；
- 可保留个人信息和允许访问的快捷入口；
- 不显示空白大面积区域。

### 15.7 数据过期

海外库存等导入型数据应显示：

- 数据日期；
- 更新时间；
- 过期提示。

## 16. 页面交互规则

- 页面刷新不重置已打开 Tab；
- 点击 KPI、图表、待办或动态时打开对应业务 Tab；
- 已打开页面不重复创建 Tab；
- Dashboard 首页 Tab 固定不可关闭；
- 返回首页时保留当前 Dashboard 时间范围可作为建议；
- 具体状态持久化方案留后续阶段设计；
- Info Panel 第一版保持默认折叠；
- Dashboard 不在 Info Panel 内重复展示主要业务数据。

## 17. 页面边界

Dashboard 不负责：

- 创建完整统计报表；
- 自定义报表；
- 编辑单据；
- 审核单据；
- 执行验收；
- 确认入库或出库；
- 修改库存；
- 修改基础资料；
- 用户权限配置；
- 系统参数配置。

以上操作必须进入对应业务页面执行。
