## 2. 任务范围

本任务仅确认：

- PC 管理端整体框架；
- 导航结构；
- 工作区结构；
- 多标签页规则；
- 右侧信息区预留方式；

本任务不涉及具体视觉样式、业务字段、表单字段、页面代码、API、数据库、ORM、Schema、Migration、Seed、技术框架、依赖安装或 Task 4.3 正文。

## 3. PC管理端总体布局

PC 管理端采用企业级 ERP 布局，由顶部导航（Header）、左侧菜单（Sidebar）、主工作区（Main Content）和右侧信息区（Info Panel）组成。Main Content 顶部设置 Tab 多标签工作区。Footer 为可选区域，第一版不作为核心布局。

```text
Header
├── Sidebar
├── Main Content
│   ├── Tab 多标签工作区
│   └── 当前页面内容
└── Info Panel（默认折叠）
```

正式规则：

- 系统进入后默认打开首页；
- Header 和 Sidebar 在业务页面中保持固定；
- Main Content 为主要业务操作区域；
- Info Panel 第一版默认关闭并保持折叠；
- 不得默认保留空白右侧栏；
- Info Panel 收起时，Main Content 使用完整可用宽度。

## 4. Header顶部导航

Header 建议高度为 64px。

### 4.1 Header左侧

- 系统 Logo；
- 系统名称：Violin ERP Lite；
- Sidebar 展开或折叠按钮。

### 4.2 Header中间

- 预留全局搜索入口；
- 搜索对象可包括产品、SKU、采购订单、委外生产单及其他业务单据；
- 本任务只预留搜索入口，不定义搜索技术方案。

### 4.3 Header右侧

- 待办事项入口；
- 消息通知入口；
- 当前公司或当前组织信息；
- 当前用户名称；
- 用户头像或用户菜单；
- Info Panel 展开或收起按钮。

用户菜单可预留：

- 个人信息；
- 修改密码；
- 退出登录。

本任务不设计具体图标和视觉样式。

## 5. Sidebar左侧菜单

Sidebar 布局尺寸建议：

- 展开状态：260px；
- 折叠状态：72px。

首页作为独立入口显示在九个一级模块之前。Sidebar 必须保留 Task 4.1 已批准的九个一级模块：

1. 基础资料；
2. 采购管理；
3. 生产管理；
4. 库存管理；
5. 出入库管理；
6. 跨境业务；
7. 统计分析；
8. 用户权限；
9. 系统设置。

### 5.1 菜单结构规则

- 不新增一级模块；
- 不删除一级模块；
- 不修改一级模块名称；
- 不将质量验收设置为一级模块；
- 一级菜单始终保留图标位置；
- 展开状态显示图标和名称；
- 折叠状态只显示图标；
- 鼠标悬停时可提示菜单名称；
- Sidebar 支持手动展开和折叠；
- 系统应记忆用户上一次展开或折叠状态；
- 菜单根据功能权限动态显示；
- 无模块权限时不显示对应模块；
- 二级菜单采用点击展开方式；
- 第一版原则上不使用三级菜单；
- 页面入口层级应控制在两级以内。

### 5.2 一级菜单点击规则

- 一级模块只有一个默认页面时，可直接进入该页面；
- 一级模块包含多个二级页面时，展开对应二级菜单；
- 再次点击已展开菜单可收起；
- 当前选中页面在 Sidebar 中保持高亮状态。

## 6. Tab多标签工作区

Task 4.2 正式采用 Tab 多标签工作区，例如：首页｜采购订单｜委外生产单｜当前库存｜SKU 详情。

正式规则：

- 首页 Tab 固定保留且不允许关闭；
- 用户打开新的业务页面时，在 Tab 区新增标签；
- 已打开页面再次点击时，切换到已有 Tab，不重复创建；
- 当前 Tab 必须具有清晰选中状态；
- 普通 Tab 支持单独关闭；
- 支持关闭其他标签；
- 支持关闭右侧标签；
- 支持关闭全部可关闭标签；
- 页面关闭后切换到最近使用的有效标签；
- Tab 数量超出可用宽度时，支持横向滚动或折叠菜单；
- 系统可记忆当前会话中的已打开 Tab；
- 第一版是否跨登录会话永久保留 Tab，不在本任务中强制要求；
- 页面刷新时应尽量保持当前路由和当前 Tab；
- Tab 仅用于页面导航，不改变业务数据或单据状态。

## 7. Main Content主工作区

Main Content 为 PC 管理端核心业务操作区域。

### 7.1 列表页面统一结构

1. 页面标题；
2. 面包屑；
3. 页面操作工具栏；
4. 查询筛选区；
5. 数据表格；
6. 分页区域。

列表页面工具栏可包含新建、编辑、删除或作废、审核、导入、导出、刷新和更多操作。具体按钮是否显示由后续页面设计、Frozen 业务规则和权限决定。

### 7.2 详情页面统一结构

1. 页面标题；
2. 单据编号或对象名称；
3. 当前状态；
4. 主要操作区；
5. 基本信息；
6. 明细信息；
7. 关联业务；
8. 操作记录；
9. 状态流转记录。

### 7.3 编辑页面统一结构

1. 页面标题；
2. 基本信息区；
3. 明细编辑区；
4. 附加信息区；
5. 保存；
6. 提交；
7. 取消或返回。

Task 4.2 不定义任何具体业务字段。

## 8. 面包屑导航

面包屑用于显示当前页面所在位置，例如“采购管理 / 采购订单”和“生产管理 / 委外生产单 / 单据详情”。

正式规则：

- 面包屑位于页面标题附近；
- 首页不需要显示复杂面包屑；
- 面包屑用于定位，不替代 Sidebar；
- 面包屑中可点击的上级页面应支持返回；
- 不得出现与 Sidebar 模块归属不一致的路径。

## 9. 右侧信息区（Info Panel）

右侧信息区作为未来扩展区域保留。

第一版正式规则：

- 默认关闭；
- 默认折叠；
- 默认不占用 Main Content 宽度；
- 不允许页面初次进入时显示空白右侧栏；
- 用户点击 Header 右上角的信息区按钮后展开；
- 再次点击后收起；
- 系统应记忆用户上一次展开或折叠状态；
- 所有业务页面采用统一的展开和折叠行为；
- Info Panel 展开时不得覆盖关键业务按钮；
- Info Panel 收起后 Main Content 自动使用完整可用宽度。

第一版只预留以下模块位置，不开发具体功能：

- AI 助手（Reserved）；
- 最近操作（Reserved）；
- 快捷入口（Reserved）；
- 操作帮助（Reserved）；
- 系统通知（Reserved）。

第一版不开发 AI 助手，不开发右侧信息区具体业务能力。本次只确定布局和预留机制；后续接入相关功能时，不应重新改变整体页面框架。

## 10. Footer规则

Footer 第一版作为可选区域：

- 不设置固定占用高度的大型 Footer；
- 可在登录页或系统信息页显示版权信息；
- 业务页面如无必要，不显示 Footer；
- 不得挤压主要业务工作区。

## 11. 导航规则

系统登录成功后：

- 默认进入首页；
- 点击 Logo 返回首页；
- 点击首页菜单返回首页；
- 点击一级模块时，默认进入该模块的第一个有权限二级页面；
- 无权限页面不得通过菜单显示；
- 用户通过直接链接访问无权限页面时，应显示无权限提示；
- 页面切换优先通过 Tab 和 Sidebar；
- 浏览器前进、后退与系统路由应保持合理一致；
- 页面刷新不得自动跳回首页；
- 当前页面位置应通过 URL 或等效路由状态表示。

## 12. 权限与导航显示

菜单显示必须同时考虑用户角色、功能权限、仓库数据权限、店铺数据权限以及厂家及厂家仓数据范围。

正式规则：

- 无查看权限时，菜单和页面入口均不显示；
- 有查看权限但无操作权限时，页面允许只读访问；
- 操作按钮按操作权限显示；
- 跨境验收人员仅显示其有权访问的生产验收入口和厂家仓数据；
- 权限控制不能只依赖前端菜单隐藏；
- 本任务只定义页面表现原则，不设计 API 和权限技术实现。

## 13. 页面状态保留

- Sidebar 展开或折叠状态应记忆；
- 一级菜单展开状态可记忆；
- Info Panel 展开或折叠状态应记忆；
- 当前 Tab 状态在当前会话中保持；
- 列表筛选条件返回页面时可适当保留；
- 分页和排序状态返回页面时可适当保留；
- 具体持久化技术方案留到后续阶段设计。

## 14. PC响应式与分辨率原则

推荐设计基准为 1920 × 1080，最低重点支持 1440 × 900。

正式规则：

- PC 管理端主要面向桌面浏览器；
- 小于 1280px 宽度时，Sidebar 可自动进入折叠状态；
- Main Content 应优先保证表格和业务内容可用；
- Info Panel 在较窄屏幕下仍默认关闭；
- 小屏幕下不应强制同时展示 Sidebar、Main Content 和 Info Panel；
- 数据表格内容超宽时允许水平滚动；
- PC 页面不直接作为微信小程序页面复用；
- 微信小程序拥有独立移动端页面设计；
- 两端共用相同后端、数据和业务规则。

## 15. 布局尺寸建议

在不进入 Task 4.3 视觉规范的前提下，采用以下布局基准：

- Header：64px；
- Sidebar 展开：260px；
- Sidebar 折叠：72px；
- Tab 工作区：建议 40–44px；
- Main Content：占据剩余可用空间；
- Info Panel：展开宽度由后续视觉规范或详细设计确定；
- Footer：默认不占固定业务区域。

以上尺寸属于布局基准，不代表 Task 4.3 最终视觉样式。

### 16.1 PC管理端主要承担

- 复杂查询；
- 批量维护；
- 数据导入；
- 数据导出；
- 权限配置；
- 系统设置；
- 多单据并行处理；
- 多标签页面切换；
- 统计分析；
- 大型数据表格查看。

### 16.2 微信小程序主要承担

- 待办查询；
- 库存查询；
- 现场验收；
- 入库；
- 出库；
- 调拨；
- 盘点；
- 跨境现场操作；
- 轻量级业务查看。

### 16.3 双端边界规则

- PC 与微信小程序不共用同一页面布局；
- 两端共用同一后端、数据库和业务规则；
- PC 的 Tab 多标签工作区不适用于微信小程序；
- Info Panel 仅属于 PC 管理端布局设计。
