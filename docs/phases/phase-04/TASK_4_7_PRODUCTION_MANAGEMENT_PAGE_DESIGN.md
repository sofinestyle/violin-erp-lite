# Task 4.7：生产管理页面设计（Production Management Page Design）

## 1. 文档状态与设计依据

- Task：Task 4.7 生产管理页面设计
- 状态：Completed / Approved
- 当前 Phase：Phase 4 页面设计（UI / Page Design，In Progress）
- 下一任务：Task 4.8 库存管理页面设计
- Task 4.8 状态：Not Started
- 技术开发：Not Started

本任务严格依据 Task 4.1 至 Task 4.6、Frozen 业务规则、Task 2.5 状态流转设计及 Database Logical Design v1.0，只定义生产管理页面结构、信息分组、组件、状态、权限和跳转关系。

本任务不发起 Database Change Request，不新增数据库表、字段、业务对象或平行数据源，不定义 API、ORM、DDL、Schema、Migration、Seed，不编写页面代码，不安装依赖，也不创建 Task 4.8 至 Task 4.10 正文。

## 2. 核心 Frozen 映射规则

### 2.1 委外生产单不保存目标仓库

`production_orders` 不存在目标仓库字段，因此：

- 委外生产单新增页面不得设置目标仓库；
- 委外生产单编辑页面不得设置目标仓库；
- 委外生产单列表不得将目标仓库作为生产单字段；
- 委外生产单详情不得将目标仓库显示为生产单基本信息；
- 不得通过备注、扩展字段或页面临时字段变相保存目标仓库。

完工后目标仓库只在分批完工时选择，并严格映射 `production_completion_records.warehouse_id`。

### 2.2 不建立独立生产异常对象

Frozen 数据库不存在生产异常表，因此：

- 不设计独立生产异常列表或详情；
- 不设计异常编号或异常类型字段；
- 不设计异常处理人、处理方案、预计解决日期或异常状态字段；
- 不新增生产异常表、字段、页面对象或平行数据源。

特殊或异常情况只通过现有能力表达：

- 生产进度状态；
- 生产进度说明；
- 预计完工日期；
- 进度比例；
- 截至记录时完工数量；
- 报告人；
- 是否要求附件；
- 一般备注和附件能力；
- 暂停、逾期、已终止状态。

### 2.3 正式生产进度状态

生产进度状态严格使用：

- 待生产；
- 已排产；
- 生产中；
- 部分完工；
- 已全部完工；
- 暂停；
- 逾期；
- 已终止。

不得将“异常”“已关闭”“已完成”“待处理”或其他未冻结值作为生产进度状态。

### 2.4 正式完工验收状态

完工验收状态严格使用：

- 待验收；
- 部分验收；
- 验收完成；
- 返工中；
- 待处理；
- 异常已关闭。

不得自行增加其他完工验收状态。

## 3. 页面定位

生产管理负责公司委外生产业务，包括：

- 委外生产单；
- 生产单审核；
- 生产进度记录；
- 分批完工；
- 生产验收；
- 生产付款辅助记录入口或关联展示；
- 生产临期和逾期跟踪；
- 合格数量的后续入库依据；
- 厂家仓及公司仓的分批完工目标选择。

正式边界：

- 不建设内部车间排产系统、工人计件或工序级管理；
- 不建设 BOM 或 MRP；
- 生产单、进度记录和分批完工均不得直接增加库存；
- 生产验收确认只形成正式入库依据；
- 实际库存增加必须通过出入库管理；
- 页面不得新增 Frozen 文档未批准的流程、字段或数据对象。

表格主要展示：

- 生产厂家；
- 计划开始日期；
- 预计完工日期；
- SKU 数量；

- 逾期；
- 已终止。

“草稿”和“待审核”属于单据及审核维度，其余生产状态必须严格使用 Frozen 生产进度状态。状态 Tab 只改变筛选条件，不直接改变业务状态。

### 4.5 列表规则

- 点击生产单编号进入生产单详情；
- 完工、验收、合格、入库、跨境发货、付款数量及金额均由合法关联记录汇总；
- 页面不得手工修改累计值、比例或付款进度；
- 已被后续业务引用的生产单不得物理删除；
- 字段、按钮及数据按功能权限、金额权限、厂家和仓库数据权限动态显示。

## 5. 新建与编辑委外生产单

委外生产单新增和编辑使用独立页面。

### 5.1 基本信息

基本信息严格映射 Frozen 字段：

- 生产厂家；
- 单据日期；
- 计划开始日期；
- 预计完工日期；
- 币种：人民币（只读）；
- 备注；
- 审核相关公共字段只读状态或操作入口。

页面不得包含：

- 目标仓库；
- 生产类型；
- 独立异常字段；
- Frozen 字段中不存在的生产负责人字段。

生产负责人如无现有公共字段或用户字段合法映射，不得新增或变相保存。

### 5.2 生产明细

生产明细包括：

- 产品查询入口；
- SKU；
- 计划数量；
- 加工单价；
- 行金额；
- SKU 正式计量语义；
- 备注。

产品只用于辅助查找正式 SKU，不形成平行产品字段。累计完工、验收、合格、入库和跨境发货数量均为只读汇总，不在新建或编辑时直接修改。

### 5.3 汇总信息

汇总区域展示：

- 明细项数；
- 计划生产总数量；
- 金额小计；
- 总金额；
- 已付金额；
- 未付金额；
- 完工比例。

新建草稿时，已付金额按合法付款记录汇总且通常为零，完工比例及各累计执行数量为零。

### 5.4 页面操作

- 保存草稿；
- 提交审核；
- 取消；
- 返回。

### 5.5 新增与编辑规则

- 只能选择有效生产厂家和有效 SKU；
- 计划数量必须大于零；
- 加工单价、数量、金额和比例遵循 Frozen 精度与范围；
- 保存草稿不得产生库存、进度、完工、验收或入库记录；
- 提交审核后按 Frozen 状态规则限制编辑；
- 已发生完工、验收、入库或跨境发货后，不得通过编辑破坏累计关系；
- 产品、SKU 和厂家业务快照按 Frozen 规则保留；
- 编辑不得覆盖历史状态、进度、完工、验收、入库、付款或审计记录；
- 不得通过回退状态规避已发生的后续业务。

### 6.1 详情头部

详情头部展示：

- 生产单编号；

- 生产进度状态；

- 实际完工日期；
- 计划总数量；
- 累计完工数量；
- 累计验收数量；
- 累计合格数量；

- 完工比例；
- 金额及付款进度；
- 逾期提示；
- 当前状态允许且用户有权限执行的操作。

金额及付款进度仅向有权限用户显示。目标仓库不得作为生产单基本信息展示。

### 6.2 详情 Tab

详情 Tab 包括：

1. 基本信息；
2. 生产明细；
3. 生产进度记录；

### 6.3 详情操作

按状态和权限显示编辑、提交审核、审核通过、审核驳回、登记生产进度、登记分批完工、创建生产验收、查看或记录生产付款、作废、导出和打印等已批准入口。

页面只提供合法业务入口，不直接修改库存。状态历史、进度、完工、验收、入库、跨境发货、付款和审计记录不得覆盖。

## 7. 生产进度记录

生产进度页面严格映射 `production_progress_records`。

### 7.1 页面字段

- 来源生产单；
- 进度日期；
- 生产阶段；
- 进度比例；
- 截至本次记录的完工数量；
- 预计完工日期；
- 进度说明；
- 报告人姓名；
- 是否要求附件；
- 一般公共字段允许的附件入口。

生产阶段选择严格使用 Frozen 生产进度状态，不设置独立异常状态。

### 7.2 进度规则

- 截至本次记录的完工数量不得小于前一次合法累计值；
- 完工数量不得超过生产单计划数量；
- 进度比例必须符合 Frozen 精度与范围；
- 进度记录不得直接增加库存；
- 进度记录不得代替正式分批完工记录；
- 进度记录不得物理删除；
- 预计完工日期变化必须保留历史；
- 暂停、逾期和已终止原因通过进度说明记录；
- 不建立独立异常记录或平行数据源。

## 8. 分批完工

分批完工严格映射 `production_completion_records` 和 `production_completion_record_items`。

### 8.1 完工主记录

字段包括：

- 来源生产单；
- 完工批次号；
- 完工日期；
- 完工后目标仓库；
- 本次总完工数量；
- 完工记录状态；
- 备注。

目标仓库只在本页面选择，可按 Frozen 仓库规则选择公司仓、对应厂家仓或其他已批准且合法的仓库类型。不得将该值回写为生产单字段。

### 8.2 完工明细

字段包括：

- 行号；
- 原生产单明细；
- SKU；
- SKU 编码快照；
- SKU 名称快照；
- 规格快照；
- 本行完工数量；
- 业务批次号；
- 备注。

### 8.3 完工规则

- 本次完工数量必须大于零；
- 累计完工数量不得超过计划数量；
- 主记录总完工数量必须等于明细数量合计；
- 分批完工只形成生产验收来源，不直接增加库存；
- 已确认完工记录不得物理删除；
- 完工后目标仓库作为后续验收和入库依据；
- 已全部完工不等于验收完成或入库完成。

## 9. 生产验收

生产验收使用统一 `inspection_orders` 和 `inspection_order_items`，只处理委外生产来源。

### 9.1 页面信息

- 验收单号；
- 来源生产单；
- 来源完工批次；
- 验收日期；
- 验收仓库；
- 验收人员；
- 验收总数量；
- 合格总数量；
- 不合格总数量；
- 验收结果；
- 不合格处理方式；
- 验收明细；
- 可选附件或备注。

### 9.2 验收状态

完工验收状态严格使用待验收、部分验收、验收完成、返工中、待处理和异常已关闭。

### 9.3 验收规则

- 生产来源与采购来源必须严格区分；
- 验收数量不得超过合法完工数量；
- 合格、不合格及明细待处理数量必须符合 Frozen 数量规则；
- 只有合格数量可形成正常入库依据；
- 不合格和待处理数量不得成为正常可用库存；
- 验收确认不得直接增加库存；
- 实际入库必须通过出入库管理；
- 已确认验收记录不得覆盖或物理删除；
- 附件为可选辅助资料，不作为验收必填条件。

## 10. 跨境验收人员规则

跨境验收人员：

- 只查看授权生产厂家；
- 只查看授权厂家仓；
- 只查看授权生产单和完工批次；
- 可登记生产验收；
- 不得修改或审核生产单；
- 不得修改生产进度汇总；
- 不得执行正式入库或修改库存；
- 不得访问无关采购、财务、用户权限和系统设置。

无权限记录不得通过搜索、URL 或跨页面跳转访问。页面应显示当前授权厂家及厂家仓范围。

## 11. 生产付款关联展示

生产付款记录严格使用现有 `production_payments`，生产管理只提供受权入口和关联展示。

可展示付款编号、生产单、生产厂家、付款日期、币种、付款金额、付款方式、银行参考号、收款账户快照、付款状态和备注。

正式规则：

- 生产付款只作为辅助记录，不替代财务系统；
- 付款状态独立于生产进度、验收和入库状态；
- 金额及敏感信息按权限显示；
- 已确认付款记录不得随意修改或物理删除；
- 本任务不新增付款表、字段或完整财务能力。

## 12. 生产执行跟踪

执行跟踪展示：

- 待生产；
- 已排产；
- 生产中；
- 部分完工；
- 已全部完工；
- 暂停；
- 逾期；
- 已终止；
- 待验收完工批次；
- 待入库合格数量。

所有数据必须来自生产单、生产单明细、生产进度记录、分批完工记录、验收记录、入库记录、跨境发货记录和付款记录。不得建立独立执行汇总数据源，不得手工修改汇总数据。

筛选可包括生产厂家、生产进度状态、审核状态、完工验收状态、入库状态、付款状态、计划与预计日期范围以及是否逾期。点击指标进入对应生产单或关联记录。

## 14. 页面状态

生产管理页面必须覆盖：

- 正常；
- 加载中；
- 无生产单；
- 无进度记录；
- 无完工记录；
- 无待验收记录；
- 无权限；
- 加载失败；
- 并发冲突；
- 数量校验失败；
- 待生产；
- 已排产；
- 生产中；
- 部分完工；
- 已全部完工；
- 暂停；
- 逾期；
- 已终止。

提交失败时保留用户已输入且仍合法的内容，并明确错误字段、原因及修正方式。并发冲突时必须重新读取最新业务状态，不得静默覆盖。业务状态不得使用未冻结值。

## 15. 权限规则

生产管理至少区分：

- 生产单查看；
- 生产单新增；
- 生产单编辑；
- 生产单提交；
- 生产单审核；
- 生产进度查看；
- 生产进度登记；
- 分批完工查看；
- 分批完工登记；
- 生产验收查看；
- 生产验收执行；
- 生产付款查看；
- 金额查看；
- 数据导出。

正式规则：

- 页面、按钮、字段和数据按授权范围动态显示；
- 制单与审核遵循 Frozen 权限分离规则；
- 跨境验收人员只访问授权厂家和厂家仓；
- 无厂家或仓库数据权限时不得查看对应数据；
- 无金额权限时隐藏生产金额及付款信息；
- 前端隐藏不替代服务端权限校验；
- 不新增“生产异常处理权限”；
- 本任务只定义页面权限表现，不新增角色或权限数据结构。

## 16. 页面跳转关系

- 生产单可跳转生产厂家、SKU、生产进度、分批完工、生产验收、入库记录、厂家仓库存、跨境发货、付款记录和操作审计；
- 生产进度可跳转来源生产单；
- 分批完工可跳转来源生产单及后续生产验收；
- 生产验收可跳转来源生产单、来源完工批次、后续入库及不合格处理；
- 付款记录可跳转来源生产单和生产厂家；
- 已打开页面优先复用现有 Tab；
- 所有跳转必须遵循功能、金额、厂家及仓库数据权限。

## 17. 页面边界

生产管理不负责：

- 内部车间排产系统；
- 工序级生产管理；
- 工人计件或工时管理；
- BOM 展开或 MRP；
- 原材料领料；
- 成本核算；
- 独立生产异常对象；
- 直接修改库存；
- 仓库实际入库；
- 跨境运输跟踪；
- 海外库存管理。

上述业务不得在本任务中建立平行能力。
