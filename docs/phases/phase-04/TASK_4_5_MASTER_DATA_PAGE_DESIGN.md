# Task 4.5：基础资料模块页面设计（Master Data Page Design）

## 1. 文档状态与页面定位

- Task：Task 4.5 基础资料模块页面设计
- 状态：Completed / Approved
- 当前 Phase：Phase 4 页面设计（UI / Page Design，In Progress）
- 下一任务：Task 4.6 采购管理页面设计
- Task 4.6 状态：Not Started
- 技术开发：Not Started

基础资料模块是产品、SKU、分类、品牌、计量单位、供应商、仓库、销售平台、店铺和客户相关信息的统一维护与查询入口，为采购、生产、库存、出入库、跨境和统计分析提供唯一、受控的基础数据来源。

本模块必须遵循：

- Task 4.1 页面架构设计，不新增、删除或改名一级模块；
- Task 4.2 PC 管理端布局与导航设计，使用 Header、Sidebar、Tab、Main Content，Info Panel 第一版默认折叠；
- Task 4.3 PC 管理端视觉规范设计，第一版仅支持 Light Mode；
- Task 4.4 已批准的跨页面 Tab 与权限表现原则；
- Frozen 业务规则及 Database Logical Design v1.0。

本任务只定义页面结构、信息分组、组件、状态、权限与跳转关系，不新增数据库表或字段，不定义 API、ORM、DDL、Schema、Migration、Seed、技术框架，不编写页面代码，不安装依赖，也不创建 Task 4.6 至 Task 4.10 正文。

通用页面规则：

- 基础资料列表采用页面标题、面包屑、操作工具栏、筛选区、表格和分页结构；
- 复杂新增与编辑使用独立页面，少量简单资料可在后续实现阶段按统一规范选择弹窗或独立页面；
- 编码、名称、状态和关联信息必须使用已批准的数据语义；
- 已被业务引用的基础资料不得物理删除，只能按 Frozen 规则停用；
- 停用资料保留历史引用，不得影响既有单据、库存流水和审计记录；
- 页面不得直接修改库存余额或业务单据。

## 2. 产品管理

### 2.1 产品列表

页面结构：

1. 页面标题与面包屑；
2. 新增产品、刷新及有权限时的导入入口；
3. 查询筛选区；
4. 产品表格；
5. 分页区域。

筛选条件可包括产品编码、产品名称、产品分类、品牌、产品类型和启用状态。

列表主要展示产品编码、中文名称、英文名称、产品分类、品牌、产品类型、默认单位、启用状态和更新时间。操作列按权限显示查看、编辑、停用或启用。

正式规则：

- 产品编码和产品名称进入对应产品详情；
- 产品编码沿用现有编码体系，不建立第二套平行编码；
- 被业务引用的产品不得物理删除；
- 停用产品仍可在历史单据中查看，但不得作为新业务的可选项；
- 产品列表不直接展示或修改库存余额。

### 2.2 产品详情

产品详情使用独立 Tab，包含：

- 基本信息：产品编码、中文名称、英文名称、产品类型和状态；
- 分类与品牌：所属分类、品牌；
- 计量信息：默认单位；
- 说明信息：产品说明；
- 关联 SKU：当前产品下的 SKU 列表及状态；
- 供应关系：已批准的产品与供应商关系；
- 生产关系：已批准的产品与生产厂家关系；
- 操作信息：创建、更新及允许展示的审计信息。

关联 SKU、供应商和生产厂家均提供有权限时的详情跳转。产品详情为资料查看入口，不提供库存调整或业务单据编辑能力。

### 2.3 产品新增

产品新增使用独立页面，按以下区域组织：

- 基本信息；
- 分类与品牌；
- 产品类型与默认单位；
- 说明信息。

正式规则：

- 必填项、可选项和取值范围以 Frozen 数据库字段语义为准；
- 产品编码必须符合既有编码规则，并按不区分大小写原则判重；
- 分类、品牌和默认单位只允许选择当前有效资料；
- 保存成功后进入产品详情或返回产品列表；
- 新增产品不得同时创建平行 SKU、库存或业务记录。

### 2.4 产品编辑

产品编辑使用独立页面，布局与新增页面保持一致。

正式规则：

- 只允许编辑获授权且未被 Frozen 规则限制的资料；
- 已被业务引用后，可能影响历史识别的编码与关键业务语义不得随意改写；
- 停用操作必须明确说明对新业务选择的影响；
- 保存前提示未保存变更，保存后返回详情并刷新展示；
- 编辑产品不得覆盖历史单据中的产品快照。

## 3. SKU 管理

### 3.1 SKU 列表

页面结构与产品列表一致。筛选条件可包括 SKU 编码、SKU 名称、所属产品、产品分类、品牌、规格信息和启用状态。

列表主要展示 SKU 编码、SKU 名称、所属产品、规格摘要、计量单位、安全库存数量、启用状态和更新时间。价格信息仅向具备相应权限的用户显示。

正式规则：

- SKU 是库存和业务单据使用的正式产品粒度；
- 不建立单把小提琴序列号入口；
- 套装作为独立 SKU 整体管理，不在列表中拆分扣减内部配件；
- SKU 编码沿用现有编码体系，不建立平行编码；
- 列表可跳转当前库存，但不得直接修改库存。

### 3.2 SKU 详情

SKU 详情使用独立 Tab，包含：

- 基本信息：SKU 编码、名称、所属产品和状态；
- 规格信息：尺寸、颜色、综合规格、材质和条码；
- 计量与价格：计量单位、已批准的默认价格信息；
- 库存规则：安全库存数量；
- 关联入口：当前库存、分仓库存、库存流水和库存预警；
- 操作信息：创建、更新及允许展示的审计信息。

库存相关区域只提供受权查看与跳转，不在 SKU 详情中直接调整库存。

### 3.3 SKU 新增

SKU 新增使用独立页面，按基本信息、所属产品、规格信息、计量与价格、安全库存和其他说明分组。

正式规则：

- SKU 必须关联一个有效产品；
- SKU 编码按不区分大小写原则判重；
- 计量单位必须使用受控有效值；
- 安全库存数量不得为负；
- 创建 SKU 不自动生成库存余额或库存流水；
- 本期不录入单琴序列号和单琴质量档案。

### 3.4 SKU 编辑

SKU 编辑使用独立页面，布局与新增页面一致。

正式规则：

- 已被业务引用的 SKU 不得物理删除；
- 影响历史识别的编码和所属关系不得随意改写；
- 价格等敏感信息按权限显示和编辑；
- 停用 SKU 后不得用于新单据，但历史单据和库存记录继续可追溯；
- 编辑 SKU 不得直接改变库存数量或历史流水。

## 4. 产品分类

产品分类页面采用左侧分类树与右侧详情或列表区域，也可在宽度不足时切换为统一树形表格。

页面展示分类编码、分类名称、上级分类、分类层级、显示顺序、说明和启用状态。

正式规则：

- 支持多级分类；
- 不允许形成循环层级；
- 新增或编辑时，上级分类只能选择有效且不会形成循环的分类；
- 分类排序只影响显示顺序，不改变业务关系；
- 被产品引用的分类不得物理删除，只能停用；
- 点击分类可查看其直接下级和关联产品。

## 5. 品牌管理

品牌管理页面采用筛选区、品牌列表和维护入口。

列表展示品牌编码、中文品牌名、英文品牌名、说明、启用状态和更新时间。支持按编码、名称和状态筛选。

正式规则：

- 品牌编码按不区分大小写原则判重；
- 被产品引用的品牌不得物理删除，只能停用；
- 停用品牌不再作为新产品可选项；
- 点击品牌可查看品牌详情及关联产品；
- 品牌维护不得改变历史业务快照。

## 6. 计量单位

计量单位页面用于统一维护产品默认单位和 SKU 计量单位的受控可选值。

页面包含单位值列表、使用状态、引用情况、筛选和受权维护入口。

正式规则：

- 计量单位必须复用项目受控字典能力，不建立第二套平行单位数据源；
- 本任务不新增独立计量单位数据库表或字段；
- 已被产品或 SKU 使用的单位值不得直接删除；
- 停用单位仍保留历史显示，但不得用于新产品或 SKU；
- 具体持久化映射和技术实现留待后续具备相应范围的阶段确定，不得改变 Frozen 数据库语义。

## 7. 供应商管理

### 7.1 供应商列表

供应商列表包含页面标题、操作工具栏、筛选区、表格和分页。

筛选条件可包括供应商编码、供应商名称、简称、联系人、结算方式和启用状态。列表主要展示供应商编码、全称、简称、联系人、联系电话、结算方式、启用状态和更新时间。

敏感银行、税务及联系信息只向具备相应权限的用户显示，并按统一脱敏规则处理。

### 7.2 供应商详情

供应商详情使用独立 Tab，包含：

- 基本信息；
- 联系信息；
- 结算与付款条件；
- 税务与银行信息；
- 关联产品及采购关系；
- 关联采购订单、采购付款和采购退货入口；
- 状态及操作信息。

正式规则：

- 业务关联区域只提供受权查看和跳转；
- 供应商详情不直接创建采购付款或修改采购单据；
- 敏感信息按字段权限脱敏，不得在无权限页面泄露；
- 被业务引用的供应商不得物理删除。

### 7.3 供应商编辑

供应商编辑使用独立页面，按基本信息、联系信息、结算信息、税务银行信息和备注分组。

正式规则：

- 供应商编码按不区分大小写原则判重；
- 敏感字段的查看和编辑权限分别控制；
- 保存敏感字段变更必须保留审计记录；
- 停用供应商后不得用于新采购业务，但历史业务继续可查看；
- 编辑不得覆盖已生成单据中的供应商快照。

## 8. 仓库管理

仓库管理使用统一仓库列表，通过仓库类型筛选或视图切换展示不同仓库，不建立平行仓库页面或数据源。

### 8.1 公司仓

- 展示公司实际仓库；
- 当前规则下公司实际仓数量为一个；
- 页面可查看仓库基本信息、授权用户范围及库存入口；
- 不提供库位、货架或层位维护。

### 8.2 厂家仓

- 每个生产厂家建立独立厂家暂存仓；
- 厂家仓必须显示并关联对应生产厂家；
- 不同厂家的库存不得合并；
- 用户只可查看被授权的厂家及厂家仓。

### 8.3 在途仓

- 在途仓用于表达调拨和跨境发货在调出后、调入前的库存节点；
- 页面展示仓库基本信息及在途库存查看入口；
- 不提供将待入库数量直接计入正式库存的操作；
- 不得以独立在途数量字段形成第二套在途库存。

### 8.4 海外仓

- 海外仓必须显示国家信息；
- 海外库存通过 Excel 定期导入，本期不提供海外平台或海外仓 API 配置；
- 页面显示最近数据日期、更新时间及必要的过期提示；
- 仓库资料页面不直接编辑导入后的库存数量。

### 8.5 仓库详情

仓库详情使用独立 Tab，包含：

- 基本信息：仓库编码、名称、类型和状态；
- 责任主体：责任主体类型及适用的生产厂家；
- 地域与联系信息；
- 库存属性：是否允许形成可用库存；
- 权限信息：当前用户可见的授权摘要；
- 关联入口：分仓库存、库存流水、库存预警、调拨及相关业务列表；
- 操作信息。

正式规则：

- 仓库类型必须保持公司仓、厂家仓、在途仓、海外仓和待处理仓的 Frozen 语义；
- 厂家仓必须关联生产厂家，海外仓必须具备国家信息；
- 已被业务引用的仓库不得物理删除；
- 仓库资料维护不得直接改变库存余额；
- 用户只能访问授权仓库数据。

## 9. 销售平台管理

销售平台管理页面包含筛选、平台列表、详情查看和受权维护入口。

列表展示平台编码、平台名称、平台类型、所属国家、是否跨境、启用状态和更新时间。

正式规则：

- 平台编码按不区分大小写原则判重；
- 被店铺或销售出库引用的平台不得物理删除；
- 停用平台不再用于新店铺或新销售出库；
- 平台详情可跳转其关联店铺；
- 本期不在本页面配置海外平台 API 对接。

## 10. 店铺管理

店铺管理页面包含筛选区、店铺列表、详情查看和受权维护入口。

筛选条件可包括店铺编码、店铺名称、所属平台、经营国家和启用状态。列表展示店铺编码、店铺名称、所属平台、外部店铺标识、经营国家、业务币种、运营负责人、启用状态和更新时间。

正式规则：

- 店铺必须关联有效销售平台；
- 店铺编码按不区分大小写原则判重；
- 外部店铺标识按所属平台范围判重；
- 用户仅可查看角色授权的店铺；
- 被销售出库引用的店铺不得物理删除；
- 店铺详情可跳转授权范围内的销售出库记录；
- 本页面不建设完整销售订单生命周期。

## 11. 客户资料

客户资料页面用于在授权范围内查询销售出库中已批准保存的客户姓名、收件国家和收件地址等客户快照，并提供来源业务追溯。

正式规则：

- 本期不建立独立客户主数据实体、客户生命周期或完整销售订单；
- 本任务不新增客户数据库表、字段或平行客户数据源；
- 客户资料来源必须是已批准的销售出库客户快照；
- 页面以查询和追溯为主，不提供独立客户新增、编辑或删除；
- 客户姓名和地址属于个人信息，必须按权限显示、脱敏并保留访问审计；
- 用户只可查看授权店铺和销售范围内的数据；
- 点击记录进入对应销售出库详情，不在客户资料页面修改历史快照。

组件视觉和交互必须遵循 Task 4.2 与 Task 4.3。本任务不选择或创建具体前端组件库。

## 13. 页面状态

基础资料页面必须覆盖：

- 正常：列表、详情或表单完整展示；
- 启用：资料可用于新业务；
- 停用：资料保留历史但不得用于新业务；
- 编辑中：存在未保存变更；
- 保存中：保存操作进行中，防止重复提交；
- 保存成功：更新当前页面并提供明确反馈；
- 无权限：不显示无权访问的资料或操作；
- 无数据、加载中、部分失败和整体异常。

状态表达不得只依靠颜色，必须同时提供文字或图标。

## 14. 权限规则

- 菜单、页面、详情、字段和操作按钮根据功能权限动态显示；
- 无查看权限时不显示对应页面入口；
- 有查看权限但无维护权限时，页面为只读；
- 产品、SKU、分类、品牌、计量单位和供应商维护权限分别控制；
- 仓库页面同时执行功能权限和仓库数据权限；
- 厂家仓页面同时执行授权厂家及厂家仓范围；
- 店铺和客户资料页面同时执行店铺数据权限；
- 银行、税务、价格、个人姓名及地址等敏感信息执行字段级查看和脱敏规则；
- 启用、停用及敏感资料变更必须保留操作记录；
- 权限控制不得只依赖前端隐藏；
- 本任务只定义页面表现，不定义权限技术实现。

## 16. 空状态

- 列表无数据时显示明确说明和当前用户有权限使用的新增入口；
- 筛选无结果时提示调整或清除筛选条件；
- 详情无关联 SKU、供应商、库存或业务记录时显示“暂无相关数据”；
- 无权限时不得使用“暂无数据”掩盖权限限制，应显示明确无权限说明；
- 客户资料无销售出库快照时只显示空状态，不创建独立客户资料；
- 空状态不得展示虚假示例数据或错误占位值。

## 17. 加载状态

- 列表首次加载使用表格骨架屏；
- 详情页按信息分组进行局部加载；
- 树形分类与关联资料可独立加载，不阻塞页面其他区域；
- 保存时禁用重复提交，并保留明确进度反馈；
- 页面切换和 Tab 状态遵循 Task 4.2；
- 长时间加载必须显示说明或重试入口，不得只显示无解释的 Spinner。

## 18. 异常状态与正式结论

异常状态规则：

- 列表加载失败时保留已成功加载的数据并提供重试；
- 详情部分区域失败时，其余区域继续显示；
- 保存校验失败时在对应字段下方显示原因，并保留用户已输入内容；
- 编码重复、关联资料失效或权限变化时给出明确原因与处理建议；
- 并发更新或资料已停用时不得静默覆盖，应提示刷新后重新处理；
- 网络异常不得造成重复新增或重复保存；
- 直接访问无权限页面时显示无权限状态；
- 异常处理不得绕过 Frozen 业务规则或直接修改数据库与库存。
