# Task 4.10：跨境业务页面设计（Cross-border Business Page Design）

## 1. 文档状态与设计依据

- Task：Task 4.10 跨境业务页面设计
- 状态：Completed / Approved
- 所属 Phase：Phase 4 页面设计（UI / Page Design，Completed / Approved）
- 下一阶段：Phase 5 接口设计
- Phase 5 状态：Waiting / Not Started
- 技术开发：Not Started

本任务严格依据 Task 4.1 至 Task 4.9、Frozen 业务规则、Task 2.5 状态流转设计及 Database Logical Design v1.0，只定义跨境业务页面结构、信息分组、组件、状态、权限和跳转关系。

本任务不新增数据库表、字段、业务对象、状态值或平行数据源，不定义 API、ORM、DDL、Schema、Migration、Seed，不编写页面代码，不安装依赖，也不创建 Phase 5 正文。

## 2. 页面定位

跨境业务页面负责：

- 跨境发货单及明细；
- 发货批次管理；
- 来源仓至在途仓发运；
- 在途运输信息展示；
- Excel 导入结果与海外实收差异展示；
- 海外数据导入；
- 导入校验；
- 发货明细匹配；
- 跨境业务执行跟踪；
- 海外仓正式库存结果查询和跳转。

跨境业务页面不负责独立跨境采购订单、独立跨境销售订单、完整物流轨迹平台、报关申报、清关系统、运费及关税核算、完整财务核算、直接修改库存余额或无来源库存流水。

## 3. 核心 Frozen 映射规则

### 3.1 跨境发货单

跨境发货业务严格映射 `cross_border_shipments` 和 `cross_border_shipment_items`。

跨境发货主记录正式字段包括：

- 可选生产单；
- 来源仓；
- 在途仓；
- 目的海外仓；
- 发货批次号；
- 承运商；
- 物流单号；
- 运输方式；
- 发运日期；
- 预计到达日期；
- 实际到达日期；
- 目的国家；
- 发货总数量；
- 发货状态；
- 单据公共字段。

跨境发货明细正式字段包括：

- 跨境发货单；
- 可选生产单明细；
- SKU 及正式快照；
- 批次号；
- 发货数量；
- 海外实收数量；
- 差异数量；
- 单位成本；
- 行成本；
- 明细公共字段。

不得新增独立跨境销售订单、货柜表、装箱单表、物流轨迹表、清关表、海外库存余额表，或 Frozen 中不存在的贸易、报关和费用字段。

### 3.2 发货库存流程

跨境发货的库存路径严格为：

来源仓
→ 在途仓

正式规则：

- 正式发运后，来源仓库存减少、在途仓库存增加；
- 跨境发运只负责来源仓库存减少、在途仓库存增加；
- 在途仓库存转为海外仓库存只能由 Task 5.5 正式 Excel 导入结果形成；
- 每次库存变化必须形成来源可追溯的正式库存流水；
- 不得从来源仓直接瞬时增加海外仓库存；
- 不得新增或使用独立 `in_transit_quantity`；
- 在途库存以在途类型仓库中的 `inventories` 正式记录为准。

### 3.3 海外实收与差异

海外实收必须由 Task 5.5 正式 Excel 导入结果形成，并在跨境发货明细中分别展示发货数量、实收数量和差异数量。差异必须保留并可追溯，不得通过修改原发货数量掩盖差异，不得将发货数量自动视为实收数量，也不得通过本页面手工增加海外仓库存。

### 3.4 导入任务

海外仓、平台或物流数据导入严格映射 `import_tasks` 和 `import_task_items`。

导入任务主记录包括任务编号、导入类型、原文件名、文件存储引用、适用的目标仓库或目标店铺、任务状态、总行数、成功行数、失败行数、警告行数、开始时间、完成时间和错误摘要。

导入任务明细包括文件行号、原始行数据、匹配 SKU、匹配仓库、校验状态、执行状态、错误代码、错误信息、结果单据类型、结果单据和处理时间。

不得新增第二套跨境 Excel 导入表、临时正式数据源或无法追溯来源的导入结果。文件本体不得直接保存在数据库中，只保存正式文件元数据和存储引用。

### 3.5 发货导入匹配

跨境发货与海外实收导入匹配严格映射 `shipment_import_matches`，正式字段包括跨境发货单、跨境发货明细、导入任务、导入任务明细、匹配数量、实收数量、差异数量、匹配状态、匹配时间、匹配用户和备注。

必须采用明细级匹配，不得只在主单级匹配，不得合并不同 SKU 或批次，不得覆盖原发货明细，也不得静默忽略无法匹配的导入行。

### 3.6 海外库存

海外库存严格映射 `warehouses`、`inventories` 和 `inventory_transactions`。跨境页面只展示正式海外仓库存结果或提供跳转，不建立独立海外库存余额数据源。

### 3.7 正式状态集合

跨境发货只使用 Frozen 已有的三个正式状态字段，不建立独立物流状态、海外收货状态或差异处理状态：

- `status`：通用单据状态；
- `approval_status`：审核状态；
- `shipment_status`：发货执行状态；
- 海外库存导入任务状态：待上传、待校验、校验失败、待确认、导入中、部分成功、导入成功、已取消、重复文件；
- 仓库类型至少包括公司仓、厂家仓、海外仓、在途仓和待处理仓。

物流信息仅展示承运商、运单号、发运日期和预计到达日期，不形成独立物流状态。

`validation_status`、`execution_status` 和 `match_status` 是 Frozen 正式字段，但现有 Approved/Frozen 文档未列出其完整枚举代码集合。本任务只以受控字段呈现校验、执行和匹配结果，并将未匹配、部分匹配、匹配异常、执行失败等作为页面语义提示；不得将这些提示擅自写成新的数据库枚举值。完整代码映射必须等待后续正式规格确认，且不得改变 Frozen 字段语义。

## 4. 跨境发货列表

页面结构包括页面标题与面包屑、状态 Tab、操作工具栏、筛选区、发货单表格和分页。

筛选条件包括：

- 发货单号；
- 发货批次号；
- 来源仓；
- 在途仓；
- 目的海外仓；
- 生产单；
- 承运商；
- 物流单号；
- 目的国家；
- 发货执行状态；
- 发运日期范围；
- 预计到达日期范围；
- 是否逾期；
- 是否存在差异；
- 创建人。

表格展示发货单号、发货批次号、来源仓、在途仓、目的海外仓、目的国家、承运商、物流单号、SKU 数量、发货总数量、海外实收总数量、差异总数量、发运日期、预计到达日期、实际到达日期、正式状态和操作。

SKU 数量、海外实收总数量和差异总数量均由 `cross_border_shipment_items` 实时汇总，不新增主表字段。状态 Tab 和筛选器只使用 `status`、`approval_status` 和 `shipment_status`，不提供独立物流、海外收货或差异处理状态筛选。

## 5. 新建与编辑跨境发货单

新建与编辑使用独立页面。

### 5.1 基本信息

基本信息包括可选生产单、来源仓、在途仓、目的海外仓、发货批次号、承运商、物流单号、发运日期、预计到达日期、目的国家和备注。物流信息仅包含承运商、物流单号、发运日期和预计到达日期。

实际到达日期在创建发货阶段只读且为空；如 Task 5.5 正式 Excel 导入执行结果依法形成该事实，只能由该结果更新，跨境页面不得手工填写。

### 5.2 发货明细

发货明细包括 SKU、可选生产单明细、批次号、来源仓可用数量、本次发货数量、单位成本、行成本、已实收数量、差异数量和备注。

来源仓可用数量为 `inventories.available_quantity` 的实时只读值；已实收数量和差异数量映射正式明细字段并在创建阶段只读。

### 5.3 正式规则

- 来源仓必须为用户有操作权限的合法仓库；
- 在途仓必须为正式在途类型仓库；
- 目的仓必须为正式海外仓；
- 来源仓、在途仓和目的仓必须两两不同；
- 发货数量必须大于零且不得超过来源仓合法可用数量；
- 批次业务必须选择合法批次；
- 关联生产单时，生产单与生产单明细必须合法且相互一致；
- 保存草稿和提交审核不得修改库存；
- 正式发运后才执行来源仓至在途仓的库存变化；
- 已发运记录不得通过编辑原发货数量掩盖后续差异；
- 已产生海外实收后不得自由修改仓库、批次、SKU、发货数量等关键字段。

## 6. 跨境发货详情

详情头部展示发货单号、发货批次号、正式状态、来源仓、在途仓、目的海外仓、目的国家、承运商、物流单号、发货总数量、实收总数量、差异总数量、发运日期、预计到达日期、实际到达日期和逾期提示。

详情 Tab 包括：

1. 基本信息；
2. 发货明细；
3. 来源生产单；
4. 发运库存流水；
5. 海外实收视图；
6. 导入匹配记录；
7. 海外入库流水；
8. 状态历史；
9. 操作审计。

“海外实收视图”由跨境发货明细和匹配记录组成，不新增独立海外收货记录表。

操作按钮按正式状态与权限显示编辑、提交审核、审核通过、审核驳回、确认发运、查看导入结果、查看匹配结果、取消或作废、导出和打印。取消或作废仅在 Frozen 状态规则允许时显示。不得显示手工海外收货、手工登记实收或手工增加海外仓库存按钮。

## 7. 发运确认

发运确认页面展示来源仓、在途仓、发货明细、批次、发货数量、来源仓当前可用数量、发运日期、操作人和确认提示。

正式确认后：

- 来源仓库存减少；
- 在途仓库存增加；
- 形成方向相反、相互关联且来源可追溯的库存流水；
- 正式记录 `departure_date`；
- 发货执行状态按待发货、部分发货、已全部发货、已取消或发货异常的正式语义流转。

确认失败不得部分静默完成，页面必须明确失败明细，并确保来源仓和在途仓不存在单边库存变化。

## 8. Excel 导入结果与海外库存形成

### 8.1 正式来源

海外仓库存的唯一正式来源是 Task 5.5 的定期 Excel 导入结果。Task 4.10 不提供手工海外收货入口、手工海外收货 API 对应操作或手工增加海外仓库存能力。

### 8.2 导入结果与明细匹配

用户可查看正式导入任务、导入明细、导入结果和明细级匹配结果。未校验、校验失败、重复文件或无权限任务不得形成海外仓正式库存。

### 8.3 正式规则

- 跨境发运只减少来源仓库存并增加在途仓库存；
- 在途仓库存减少和海外仓库存增加只能由 Task 5.5 正式 Excel 导入结果形成；
- `received_quantity`、`difference_quantity` 和 `actual_arrival_date` 只展示正式导入执行结果，不允许在本页面手工录入；
- 差异数量根据正式发货数量和实收数量计算，不允许直接编辑；
- 不得修改原发货数量消除差异；
- 未完成匹配的导入行不得静默进入海外仓；
- 库存变化必须形成正式库存流水并可追溯至导入任务、导入明细和匹配记录；
- 已形成的导入结果不得物理删除或自由修改。

## 9. 海外数据导入

### 9.1 导入任务列表

筛选条件包括任务编号、导入类型、文件名、目标仓库、目标店铺、任务状态、创建日期、完成日期和创建人。

表格展示任务编号、导入类型、文件名、目标仓库或店铺、总行数、成功行数、失败行数、警告行数、任务状态、开始时间、完成时间、错误摘要和操作。

任务状态严格使用待上传、待校验、校验失败、待确认、导入中、部分成功、导入成功、已取消和重复文件。

### 9.2 新建导入任务

页面包括导入类型、文件选择、适用的目标仓库或目标店铺、模板下载、文件要求说明、开始校验和取消。

文件本体通过正式文件存储机制保存，`import_tasks` 只保存原文件名和 `file_reference`，不得保存文件二进制或建立第二套跨境导入表。

### 9.3 导入预检

文件上传后先进入预检页面，展示文件基本信息、总行数、字段识别结果、SKU 匹配结果、仓库匹配结果、成功行、警告行、失败行、错误原因和原始行数据。

允许操作包括下载错误明细、返回修改文件、重新上传、在 Frozen 字段能力允许时人工修正 SKU 或仓库匹配，以及确认执行。

正式规则：

- 校验失败行不得执行；
- 警告行必须明确展示影响；
- 不得静默跳过失败行并声称全部成功；
- `raw_data` 保留原始行数据以便审计；
- 无法匹配的 SKU 不得自动创建；
- 校验失败或重复文件不得进入执行状态；
- 执行结果必须关联正式结果单据或发货明细匹配记录；
- `validation_status` 和 `execution_status` 只使用后续正式批准的受控值，本任务不新增枚举代码。

## 10. 发货明细匹配

页面严格采用明细级匹配。

导入行区域展示文件行号、外部 SKU、外部批次、原始数量、实收数量、仓库信息、校验状态和错误信息。外部 SKU、外部批次和原始数量均从 `raw_data` 受控解析展示，不新增正式字段。

系统发货明细区域展示发货单、发货批次、SKU、批次、发货数量、已匹配数量和剩余可匹配数量。已匹配数量及剩余可匹配数量由 `shipment_import_matches` 实时汇总计算。

匹配结果区域展示匹配数量、实收数量、差异数量、匹配状态和备注。

正式规则：

- SKU 和批次必须准确匹配；
- 匹配数量不得超过导入行可匹配数量；
- 匹配数量不得超过发货明细剩余可匹配数量；
- 一个导入行拆分匹配或多个导入行对应同一明细时，必须保持数量一致；
- 不得只按主单总量匹配；
- 未匹配、部分匹配和异常匹配必须明确展示；
- 确认匹配后记录匹配用户和匹配时间；
- 匹配记录不得物理删除；
- `match_status` 只使用后续正式批准的受控值，本任务不新增枚举代码。

## 11. 跨境执行跟踪

执行跟踪以 `status`、`approval_status`、`shipment_status`、发运日期、预计到达日期及导入结果展示发运与导入执行进度，不建立物流、海外收货或差异处理状态视图。

筛选条件包括来源仓、在途仓、海外仓、目的国家、承运商、正式状态、日期范围、是否逾期和是否存在差异。

所有数据必须由跨境发货单、跨境发货明细、导入任务、导入任务明细、发货导入匹配、库存流水和正式库存实时汇总，不建立独立跨境执行汇总数据源。

## 12. 海外仓库存入口

跨境页面提供海外仓库存只读视图或跳转，展示海外仓、SKU、账面现存数量、可用数量、预留数量、待处理数量、最后流水时间，以及有权查看的来源跨境发货单、导入任务或匹配记录入口。

正式规则：

- 正式库存来源仍为 `inventories`；
- 海外仓库存只能由 Task 5.5 正式 Excel 导入结果形成；
- 不建立独立海外库存余额表；
- 不建立历史余额快照、快照差异或快照查询入口；
- 保留当前库存、库存流水、Excel 导入任务、导入明细、导入结果和来源追溯；
- 导入文件在未形成合法业务结果前不得作为正式库存展示；
- 页面不得手工修改库存；
- 来源业务通过 `inventory_transactions` 追溯，不新增库存来源字段。

## 14. 页面状态

页面必须覆盖正常、无数据、无权限、首次加载、局部加载、加载失败、库存不足、批次不足、并发冲突、来源状态变化和已被其他用户处理等界面状态。

业务状态严格使用第 3.7 节正式集合。导入成功、失败、警告、匹配和数量差异只作为导入结果或页面提示，不得建立独立物流、海外收货、差异处理状态或新增数据库状态值。

正式规则：

- 来源生产单、仓库、库存、批次或导入任务状态变化时重新读取最新数据；
- 库存或批次不足时阻止发运确认；
- 并发冲突不得静默覆盖；
- 导入部分成功必须同时展示成功、失败和警告统计；
- 加载或执行失败时保留已成功数据并提供重试或错误下载入口；
- 网络异常不得造成重复库存流水、重复匹配或重复导入结果；
- 无权限页面不得泄露仓库、成本、原始导入数据或敏感信息。

## 15. 权限规则

跨境业务至少区分：

- 跨境发货查看、创建、编辑、提交和审核；
- 发运确认；
- 海外实收与导入结果查看；
- 导入任务查看和创建；
- 导入预检和执行；
- 发货明细匹配；
- 差异查看；
- 成本查看；
- 海外仓库存查看；
- 数据导出。

正式规则：

- 用户只能访问授权来源仓、在途仓和海外仓；
- 跨境验收人员只访问授权厂家及厂家仓相关业务；
- 无成本权限时隐藏单位成本和行成本；
- 导入文件原始数据按权限和敏感级别显示；
- 制单与审核遵循 Frozen 权限分离原则；
- 无对应权限时不显示页面入口、字段或操作按钮；
- 前端隐藏不替代服务端权限校验；
- 本任务不新增角色、权限表或技术实现。

## 17. 空状态、加载状态与异常状态

### 17.1 空状态

- 无发货单、导入任务或匹配记录时显示明确说明和有权限的操作入口；
- 筛选无结果时提供清除或调整筛选条件入口；
- 无待导入或待匹配数据时不得创建临时业务对象；
- 无权限不得伪装为无数据。

### 17.2 加载状态

- 列表首次加载使用表格骨架屏；
- 详情、明细、流水和匹配区域支持局部加载；
- 文件上传、预检、执行和匹配提供明确进度；
- 提交、审核、发运和导入结果处理期间禁止重复操作；
- 长时间运行的导入任务展示当前正式任务状态，不使用无说明的加载动画。

### 17.3 异常状态

- 文件格式、必填列、SKU、海外仓、数量和重复文件错误必须逐行或按任务明确展示；
- 未匹配行、部分成功任务和导入数量差异不得静默忽略；
- 来源状态变化、权限变化或并发处理时提示刷新后重新操作；
- 单个区域失败不得清空其他已成功加载信息；
- 异常处理不得绕过在途仓、明细匹配、实收确认、库存流水或负库存规则。
