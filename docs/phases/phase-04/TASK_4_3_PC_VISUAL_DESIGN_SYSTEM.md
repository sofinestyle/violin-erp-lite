# Task 4.3：PC 管理端视觉规范设计（PC Visual Design System）

## 1. 文档状态

- Task：Task 4.3 PC 管理端视觉规范设计
- 状态：Completed / Approved
- 当前 Phase：Phase 4 页面设计（UI / Page Design，In Progress）
- 下一任务：Task 4.4 首页 Dashboard 页面设计
- Task 4.4 状态：Not Started
- 技术开发：Not Started

## 2. 文档目的

本文档在已批准的 Task 4.1 页面架构设计和 Task 4.2 PC 管理端布局与导航设计基础上，统一 PC 管理端的视觉语言与通用界面表现，为后续页面详细设计提供正式依据。

本任务只确定视觉定位、颜色、字体、间距、圆角与阴影，以及 Header、Sidebar、Tab、按钮、表单、表格、状态标签、卡片、弹窗、抽屉、图标、反馈、加载、空状态和无障碍规则。

本任务不涉及业务字段详细设计，不编写页面代码或组件，不选择具体组件库或图标库，不创建 API，不修改数据库，不创建 ORM、DDL、Schema、Migration 或 Seed，不制作 Figma 或交互原型，不开始 Task 4.4 至 Task 4.10 正文设计。

## 3. 适用范围与设计基线

- 适用于 Violin ERP Lite PC 管理端；
- 页面架构必须遵循 Task 4.1，不新增、删除或改名九个已批准一级模块；
- 页面布局与导航必须遵循 Task 4.2，不改变 Header、Sidebar、Main Content、Tab 和 Info Panel 框架；
- 页面显示文字默认使用中文（简体）；
- 页面视觉不得改变 Frozen 业务规则、Database Logical Design v1.0 或固定九阶段路线；
- 第一版本只定义 Light Mode。

## 4. 整体视觉定位

PC 管理端采用干净、专业、清晰、轻量的企业级 ERP 视觉风格，并保持适中的信息密度。

- 优先保证表格、数字、状态和操作层级清晰；
- 通过稳定的间距、边框、字号和色彩建立层级，减少无业务意义的装饰；
- 不使用大面积复杂渐变、重阴影、卡通化图形或过度圆角；
- 操作区域必须区分主操作、次操作和危险操作；
- 高密度业务页面须兼顾紧凑性与可读性；
- 第一版本只支持 Light Mode，不将 Dark Mode 纳入本期设计、开发或测试范围。

## 5. 色彩系统

### 5.1 主色

| Token | 色值 | 用途 |
| --- | --- | --- |
| Primary 600 | `#2563EB` | 主按钮、当前状态、重要交互 |
| Primary 500 | `#3B82F6` | 链接、Focus、图表辅助色 |
| Primary 100 | `#DBEAFE` | 选中背景、浅色提示背景 |
| Primary 50 | `#EFF6FF` | 轻量 Hover、弱强调背景 |

主色用于主按钮、链接、焦点状态、当前 Tab、选中菜单、主要图表和关键业务入口。不得将主色用于大面积高饱和背景。

### 5.2 中性色

| 用途 | 色值 |
| --- | --- |
| 页面背景 | `#F5F7FA` |
| 卡片背景 | `#FFFFFF` |

| 次要文字 | `#4B5563` |

| 边框 | `#E5E7EB` |

| Hover 背景 | `#F3F4F6` |

### 5.3 状态色

| 状态 | 色值 |

状态不得只通过颜色表达，必须同时使用文字、图标或形状。相同状态在全系统保持相同表达，不使用大面积高饱和状态色块。

## 6. 字体与文字层级

### 6.1 字体族

- 中文：`PingFang SC`、`Microsoft YaHei`、`Noto Sans SC`；
- 英文与数字：`Inter`、`Arial`、`sans-serif`。

### 6.2 字号规范

| 类型 | 字号 | 字重 |
| --- | --- | --- |
| 页面标题 | 24px | 600 |
| 卡片标题 | 18px | 600 |
| 区块标题 | 16px | 600 |
| 正文与表格 | 14px | 400 |
| 辅助说明 | 13px | 400 |
| 标签文字 | 12px | 400 |
| 关键数字 | 28px–32px | 600 |

默认行高为 1.5。不得使用过小或过细、影响业务信息识别的文字。

## 7. 间距、圆角与阴影

### 7.1 间距系统

采用 4px 基础间距系统，正式间距值为：4px、8px、12px、16px、24px、32px、40px。

- 页面左右内边距：24px；
- 页面上下内边距：20px–24px；
- 卡片内部间距：16px；
- 表单行间距：16px。

### 7.2 圆角

- 输入框与按钮：6px；
- 卡片与弹窗：8px；
- 状态标签：4px；
- 头像：圆形。

### 7.3 阴影

- 普通卡片不使用明显阴影，以白色背景和浅灰边框建立边界；
- 下拉菜单、弹窗和浮层可使用轻量阴影；
- 不使用重阴影或多层高对比阴影。

## 8. Header 视觉规范

Header 延续 Task 4.2 确认的 64px 高度。

- 背景使用白色，底部使用浅灰分隔线；
- Logo 与系统名称保持简洁，避免复杂装饰；
- 全局搜索采用浅灰背景；
- 功能图标统一使用线性图标；
- 用户头像尺寸为 32px；
- 图标按钮点击区域为 36px × 36px；
- Hover 使用浅灰或浅蓝背景；
- 不使用大幅缩放或复杂动画。

## 9. Sidebar 视觉规范

| 项目 | 规范 |
| --- | --- |
| 背景 | `#111827` |
| 主要文字 | `#D1D5DB` |
| 辅助文字 | `#9CA3AF` |
| 选中背景 | `#1D4ED8` |
| 选中文字 | `#FFFFFF` |
| Hover 背景 | `#1F2937` |

- 一级菜单高度为 44px，二级菜单高度为 40px；
- 菜单图标采用统一线性风格；
- 当前模块与当前页面必须有清晰选中状态；
- 二级菜单使用明确缩进；
- 折叠状态只显示图标；
- 不为不同模块配置不同主题色；
- 必须保持 Task 4.1 已批准的九个一级模块。

## 10. Tab 视觉规范

- Tab 区高度为 40px–44px；
- Tab 区域使用浅灰背景；
- 当前 Tab 使用白色背景并显示蓝色指示线；
- 非当前 Tab 使用中灰文字；
- 普通业务 Tab 允许关闭，首页 Tab 不显示关闭入口；
- 未保存状态使用小圆点提示；
- 不使用胶囊形或大圆角 Tab；
- Tab 溢出与切换行为遵循 Task 4.2。

## 11. 按钮规范

### 11.1 主按钮

用于新建、保存、提交、确认和审核等主要动作，采用蓝色背景、白色文字、高度 36px、圆角 6px。

### 11.2 次按钮

用于取消、返回、刷新和导出等次要动作，采用白色背景、灰色边框和深色文字。

### 11.3 危险按钮

用于删除、作废、驳回和取消审核等危险动作，采用红色文字或红色背景，并要求二次确认。

### 11.4 通用规则

- 同一操作区域原则上最多只有一个主按钮；
- 禁用状态必须降低对比度且不可点击；
- 纯图标按钮必须提供 Tooltip；
- 用户无对应功能权限时，不显示相关按钮或入口。

## 12. 表单规范

- 常规输入控件高度为 36px，紧凑型为 32px；
- 控件必须覆盖 Default、Hover、Focus、Error、Disabled 和 Readonly 状态；
- 必填字段使用红色星号标识；
- 错误说明显示在对应控件下方；
- 只读字段使用浅灰背景；
- 字段标签默认位于输入控件上方；
- 字段较多时必须按业务含义分组；
- 不得只依赖 Placeholder 代替字段标签；
- 错误状态不得只通过边框颜色表达，应同时显示错误文字或图标。

## 13. 表格规范

- 表头高度为 40px，标准行高为 44px，紧凑行高为 36px；
- 表头使用浅灰背景与中等字重；
- 行之间使用浅色分隔线，并提供行 Hover 状态；
- 文本左对齐，数量和金额右对齐，状态左对齐或居中；
- 操作列固定在右侧，关键编号列可固定在左侧；
- 超长内容使用省略号，并支持 Hover 查看完整内容；
- 无数据时必须展示明确空状态；
- 批量操作仅在选择数据后可用；
- 排序、筛选和分页的视觉与交互保持一致；
- 金额统一显示为类似 `¥12,345.00` 的格式。

## 14. 状态标签规范

状态标签采用浅色背景和深色文字：

- 已完成：绿色；
- 处理中：蓝色；
- 待处理：紫色；
- 异常：红色；
- 待审核：橙色；
- 已关闭：灰色。

同一状态在不同页面必须保持一致；标签文字应简短；不得使用大面积高饱和状态标签；状态含义不得只依赖颜色表达。

## 15. 卡片与面板规范

- 卡片使用白色背景、1px 浅灰边框、8px 圆角；
- 卡片内边距为 16px–24px；
- 卡片之间保持清晰间距；
- 首页 KPI 卡片可包含指标名、数值、变化信息和简洁图标；
- 同一页面不得使用过多不同颜色；
- Info Panel 第一版本保持默认折叠，只预留视觉位置，不开展具体功能设计或开发。

## 16. 弹窗与抽屉规范

### 16.1 使用场景

- 弹窗用于删除确认、审核确认、简单编辑和少量字段录入；
- 抽屉用于快速查看、辅助信息和轻量操作；
- 复杂单据编辑必须使用独立页面，不得放入弹窗。

### 16.2 尺寸

- 小型弹窗：480px；
- 中型弹窗：640px；
- 大型弹窗：800px；
- 抽屉：480px–640px。

危险操作弹窗必须清晰说明影响，并要求用户确认；不得在弹窗中承载复杂长表单。

## 17. 图标规范

- 图标采用统一线性风格；
- 常用尺寸为 16px、18px、20px；
- 描边粗细保持一致；
- 不使用彩色拟物图标；
- 相同动作在全系统使用相同图标；
- 只有图标、没有文字的操作必须提供 Tooltip；
- 具体图标库在后续具备相应范围的技术阶段确定，本任务不指定图标库。

## 18. 反馈、加载与空状态

系统必须提供成功、错误、警告、加载、空状态、无权限和网络异常等反馈。

- 保存成功使用轻量反馈；
- 删除和作废必须二次确认；
- 加载优先使用骨架屏或局部加载；
- 不得长时间显示无说明的加载图标；
- 错误反馈应说明原因并给出处理建议；
- 空状态应提供可理解的说明，并在适用时给出下一步操作或业务入口。

## 19. 无障碍与可用性

- 文字、图标和背景之间必须保持足够对比度；
- 可点击区域原则上不小于约 32px × 32px；
- 键盘焦点状态必须清晰；
- 错误和状态不得只通过颜色传达；
- 重要操作应支持键盘访问；
- 危险操作不得与主操作紧邻或使用相似视觉；
- 所有关键业务状态必须包含可读文字。

## 20. 显示模式

第一版本仅支持 Light Mode。Dark Mode 可在后续正式立项并批准后评估，但不属于本期设计、开发或测试范围；本任务不定义深色变量，也不提出深色实现要求。
