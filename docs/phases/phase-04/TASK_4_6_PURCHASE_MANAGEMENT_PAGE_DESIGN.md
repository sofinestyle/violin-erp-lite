# Task 4.6：采购管理页面设计（Purchase Management Page Design）

## 1. 文档状态与页面定位

- Task：Task 4.6 采购管理页面设计
- 状态：Completed / Approved
- 当前 Phase：Phase 4 页面设计（UI / Page Design，In Progress）
- 下一任务：Task 4.7 生产管理页面设计
- Task 4.7 状态：Not Started
- 技术开发：Not Started

采购管理页面用于统一管理国内采购订单、跨境直采订单、采购订单审核、采购执行跟踪、分批到货、采购验收、采购退货、采购付款辅助记录，以及未履约、临期和逾期事项。

本文件于 2026-07-21 根据 Task 5.2 映射检查及项目负责人正式决定修正采购类型、采购负责人、到货入口、采购与生产关系及关闭状态页面口径；修正不改变 Task 4.6 的 Completed / Approved 状态。

本任务遵循以下边界：

- 采购订单不得直接增加库存；
- 采购验收不得直接增加库存，只确认合格数量并形成合法入库依据；
- 实际库存增加必须通过出入库管理中的正式入库流程完成；
- 采购付款仅作为付款辅助记录，不替代财务系统；
- 本期不建设完整应付账款、财务核算或银行对账能力；
- 页面不得新增未经批准的业务流程、数据库对象或平行数据源；
- 页面结构、权限与交互必须遵循 Task 4.1 至 Task 4.5、Frozen 业务规则及 Database Logical Design v1.0。

## 2. 采购订单列表

### 2.1 页面结构

采购订单列表由以下区域组成：

1. 页面标题与面包屑；
2. 订单状态 Tab；
3. 操作工具栏；
4. 查询筛选区；
5. 采购订单表格；
6. 分页区域。

工具栏按权限显示新增采购订单、刷新、导出及已批准的批量操作入口。

### 2.2 筛选条件

筛选条件包括：

- 采购订单号；
- 供应商；
- 订单状态；
- 审核状态；
- 验收状态；
- 入库状态；
- 付款状态；
- 预计交货日期；
- 是否临期；
- 是否逾期；
- 创建日期。

### 2.3 表格字段

表格主要展示：

- 采购订单号；
- 供应商；
- SKU 数量；
- 采购总数量；
- 已到货数量；
- 已验收数量；
- 已入库数量；
- 订单金额；
- 已付款金额；
- 预计交货日期；
- 订单状态；
- 逾期标识；
- 创建人；
- 操作。

### 2.4 状态 Tab

状态 Tab 包括：

- 全部；
- 草稿；
- 待审核；
- 已审核；
- 已驳回；
- 已完成；
- 已取消；
- 已作废；

### 2.5 列表规则

- 状态 Tab 只改变筛选条件，不改变订单业务状态；
- 点击采购订单号进入采购订单详情；
- 字段、操作及数据范围按权限动态显示；
- 订单金额和付款金额仅向具备金额查看权限的用户显示；
- 列表中的到货、验收、入库和付款数量或金额均为关联记录汇总，不允许直接编辑；
- 批量操作只允许使用 Frozen 规则已批准的能力；
- 已被后续业务引用的采购订单不得物理删除。

## 3. 采购订单新增

采购订单新增使用独立页面，由基本信息、采购明细、汇总信息和底部操作区组成。

### 3.1 基本信息

基本信息包括：

- 供应商；
- 订单日期；
- 预计交货日期；
- 付款方式；
- 币种：人民币（只读）；
- 备注。

### 3.2 采购明细

采购明细表包括：

- 产品；
- SKU；
- 采购数量；
- 计量单位；
- 单价；
- 行金额；
- 明细预计交货日期；
- 备注。

### 3.3 汇总信息

汇总区域展示：

- 明细项数；
- 采购总数量；
- 订单总金额；
- 已付款金额；
- 未付款金额。

新建草稿时，已付款金额为关联付款记录的汇总展示，不允许在采购明细中直接修改。

### 3.4 页面操作

底部操作包括：

- 保存草稿；
- 提交审核；
- 取消；
- 返回。

### 3.5 新增规则

- 供应商和 SKU 必须为当前有效且在授权范围内的正式资料；
- 计量单位沿用 SKU 的正式计量语义；
- 采购数量必须大于零；
- 数量、单价和金额精度必须遵循 Frozen 数据规范；
- 草稿不得产生库存、验收或入库记录；
- 提交审核后，字段编辑受采购订单状态规则限制；
- 采购订单不得直接创建库存入库结果；
- 供应商、产品、SKU、价格等业务快照必须按 Frozen 规则保留；
- 页面不得新增币种选择或改变人民币单币种规则。

## 4. 采购订单编辑

采购订单编辑与新增页面保持相同的信息分组。

正式规则：

- 只允许在 Frozen 状态规则允许时编辑；
- 已进入审核、验收、入库、退货或付款流程的关键字段不得绕过状态规则修改；
- 保存前校验来源资料、数量、金额和交货日期；
- 页面离开前提示未保存变更；
- 编辑不得覆盖历史状态、审核、验收、入库、付款或审计记录；
- 页面不得通过回退状态规避已发生的后续业务。

## 5. 采购订单详情

### 5.1 详情头部

详情头部展示：

- 采购订单号；
- 当前订单状态；
- 审核状态；
- 供应商；
- 订单总金额；
- 已付款金额；
- 预计交货日期；
- 临期或逾期提示；
- 当前状态允许且用户有权限执行的操作按钮。

### 5.2 详情 Tab

详情 Tab 包括：

- 基本信息；
- 采购明细；
- 执行进度；
- 到货记录；
- 验收记录；
- 入库记录；
- 付款记录；
- 退货记录；
- 状态历史；
- 操作审计。

到货记录仅作为采购执行视图，依据采购明细累计到货数量及关联验收、入库记录展示，不建立独立到货数据源。

### 5.3 详情操作

详情页按订单状态和权限显示：

- 编辑；
- 提交审核；
- 审核通过；
- 审核驳回；
- 创建采购验收单；
- 查看到货进度；
- 创建付款记录；
- 创建采购退货；
- 作废；
- 导出；
- 打印。

### 5.4 详情规则

- 已审核及已被下游引用的关键字段按 Frozen 规则限制编辑；
- 作废必须二次确认并说明影响；
- 已被下游引用的采购订单不得物理删除；
- 页面只允许合法状态流转，不得直接调整库存；
- 状态历史、审核记录、验收记录、入库记录、付款记录和审计记录只读且不可覆盖；
- 到货展示与入口不得新增独立到货业务对象或平行数据源。

## 6. 采购执行跟踪

采购执行跟踪页面用于汇总订单履约情况。

### 6.1 进度指标

页面展示：

- 采购数量；
- 已到货数量；
- 待到货数量；
- 已验收数量；
- 合格数量；
- 不合格数量；
- 已入库数量；
- 已退货数量；
- 已付款金额；
- 未付款金额。

### 6.2 执行视图

支持以下视图：

- 未履约；
- 部分履约；
- 即将到期；
- 已逾期；
- 待验收；
- 待入库；
- 未结清。

筛选条件包括供应商、日期范围和订单状态。采购责任通过创建人、提交人、审核人及操作审计追溯，不设置独立采购负责人字段或筛选。

### 6.3 跟踪规则

- 所有执行数据均由合法关联记录汇总，不允许手工改写；
- 状态颜色遵循 Task 4.3 视觉规范；
- 点击数量或金额进入对应订单、验收、入库、退货或付款记录；
- 汇总数据必须遵循用户的功能权限、金额权限和数据权限；
- 执行跟踪不得代替正式业务单据或改变业务状态。

## 7. 采购验收

采购验收归属采购管理，用于确认采购来源货物的质量验收结果。

### 7.1 验收列表

筛选条件包括：

- 验收单号；
- 来源采购订单；
- 供应商；
- 验收状态；
- 验收人员；
- 验收日期。

表格展示：

- 验收单号；
- 采购订单号；
- 供应商；
- 待验收数量；
- 合格数量；
- 不合格数量；
- 待处理数量；
- 验收状态；
- 验收人员；
- 验收时间；
- 操作。

### 7.2 验收编辑与详情

页面展示：

- 来源采购订单；
- SKU；
- 采购数量；
- 历史已验收数量；
- 本次待验收数量；
- 合格数量；
- 不合格数量；
- 待处理数量；
- 不合格原因；
- 处理方式；
- 备注；
- 可选附件；
- 验收人员；
- 验收时间。

不合格处理方式包括退货、换货、补货、待处理及其他符合 Frozen 规则的方式。附件为可选辅助资料，不作为采购验收必填条件。

页面操作包括保存草稿、确认验收、返回编辑和查看来源采购订单。

### 7.3 验收规则

- 本次合格、不合格与待处理数量合计不得超过本次合法待验收数量；
- 累计验收数量不得超过合法到货数量；
- 验收确认只形成正式入库依据，不直接增加库存；
- 不合格和待处理数量不得成为可用库存；
- 已确认验收记录及历史明细不可覆盖或删除；
- 验收权限通过角色权限及数据权限控制，无权限时不显示入口；
- 采购验收只显示采购来源，不显示委外生产来源。

## 8. 采购退货

采购退货必须关联来源采购订单、来源入库记录和 SKU，并可追溯来源采购验收及合法可退数量。

### 8.1 退货列表

列表展示：

- 采购退货单号；
- 来源采购订单；
- 供应商；
- SKU 数量；
- 退货数量；
- 退货原因；
- 处理方式；
- 状态；
- 退货日期；
- 操作。

### 8.2 退货处理

处理方式包括：

- 退款；
- 补货；
- 换货；
- 其他。

### 8.3 退货规则

- 采购退货必须通过实际退货出库完成库存减少；
- 创建采购退货单不得直接减少库存；
- 采购退货明细必须关联正式来源入库明细；
- 退货数量不得超过来源记录的合法可退数量；
- 已完成退货记录不可自由修改且不得物理删除；
- 退货原因和处理方式必须完整记录；
- 退货页面不得绕过库存、出库和审计规则。

- 付款金额及累计规则必须遵循 Frozen 金额规则；
- 已确认付款记录不得自由修改或删除；
- 付款金额、付款方式、凭证和供应商敏感资料按权限显示；
- 采购订单详情汇总展示付款进度，但不得在订单表格中直接改写；
- 本期不建设完整应付账款、账龄分析、银行对账或财务核算。

## 11. 页面状态

采购管理页面必须覆盖：

- 正常状态；
- 首次加载；
- 局部加载；
- 无采购订单；
- 无验收记录；
- 无付款记录；
- 无权限；
- 加载失败；
- 来源单据状态已变化；
- 并发修改冲突；
- 数量或金额校验失败；
- 已作废；
- 已逾期。

加载或提交失败时应保留用户已输入且仍合法的内容，明确指出错误字段、原因和修正方式。来源状态变化或并发冲突时必须重新读取最新状态，不得静默覆盖。

## 12. 权限规则

采购管理至少区分：

- 采购订单查看；
- 采购订单新增；
- 采购订单编辑；
- 采购订单提交；
- 采购订单审核；
- 采购订单作废；
- 采购验收查看；
- 采购验收执行；
- 采购退货查看；
- 采购退货创建；
- 采购付款查看；
- 采购付款记录；
- 金额查看；
- 数据导出。

正式规则：

- 页面、按钮、字段和数据根据用户授权范围动态显示；
- 制单与审核遵循 Frozen 权限分离规则；
- 无金额查看权限时隐藏订单、单价、付款等金额信息；
- 供应商及采购数据仅在授权数据范围内显示；
- 前端隐藏不替代服务端权限校验；
- 本任务只定义页面权限表现，不新增角色、权限表或权限业务流程。

## 13. 页面跳转关系

- 采购订单可跳转供应商、SKU、采购验收、入库记录、付款记录、采购退货和操作审计；
- 采购验收可跳转来源采购订单、后续入库、不合格处理和采购退货；
- 采购退货可跳转来源采购订单、来源验收和实际退货出库；
- 采购付款可跳转来源采购订单和供应商；
- 已打开页面优先复用现有 Tab，避免重复打开；
- 所有跳转均受功能权限、金额权限和数据权限控制，无权限时不显示入口。

## 14. 页面边界

本任务不提供：

- 直接修改库存余额；
- 仓库实际入库操作；
- 采购退货实际出库操作；
- 完整财务核算；
- 应付账款账龄；
- 银行对账；
- 供应商 CRM；
- 委外生产管理；
- 海外运输管理。

上述业务必须进入已批准的对应模块或等待相应阶段与任务，不得由采购页面建立平行业务能力。
