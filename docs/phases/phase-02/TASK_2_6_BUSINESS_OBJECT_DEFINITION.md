# Task 2.6 业务对象定义（Business Object Definition）

## 1. 文档目的与范围

本文档在已批准的 Task 2.4 业务生命周期和 Task 2.5 状态体系基础上，统一定义 Violin ERP Lite 的基础对象、业务对象、库存对象和系统对象，为后续阶段提供明确的业务语义与对象边界。

本文档只进行业务对象定义，不定义数据库字段、主键、外键、字段类型、API、页面或技术实现。对象的“生命周期”和“状态”均为业务语义；已有正式定义时直接引用 Task 2.4 与 Task 2.5，未单独定义时不在本任务新增状态枚举。

## 2. 统一对象模板

每个对象统一说明中文名称、英文名称、定义、作用、生命周期、状态、关联对象和输出。对象之间的关联仅表达业务关系，不表达数据关系或技术实现。

## 2.6.1 基础对象

### BO-B01 产品

1. **中文名称**：产品
2. **英文名称**：Product
3. **定义**：描述一类可经营乐器或配件的统一主数据，不直接代表具体库存规格。
4. **作用**：统一产品名称、分类、品牌和通用属性，避免同一产品形成平行档案。
5. **生命周期**：提出新增需求、资料确认、审核启用、业务引用、停用并保留历史。
6. **状态**：引用 Task 2.5 的产品状态体系。
7. **关联对象**：SKU、产品分类、品牌、供应商、生产厂家。
8. **输出**：经审核启用的标准产品档案及可供业务引用的产品信息。

### BO-B02 SKU

1. **中文名称**：SKU
2. **英文名称**：Stock Keeping Unit
3. **定义**：产品在确定规格组合下的最小库存核算单位。
4. **作用**：作为采购、生产、验收、库存、出入库和跨境业务的统一数量管理对象。
5. **生命周期**：随产品建立、规格确认、审核启用、业务引用、停用并保留历史。
6. **状态**：引用 Task 2.5 的 SKU 状态体系。
7. **关联对象**：产品、库存、库存流水、各类正式业务单据。
8. **输出**：唯一且可引用的 SKU 档案；本期不形成单把小提琴唯一序列号。

### BO-B03 产品分类

1. **中文名称**：产品分类
2. **英文名称**：Product Category
3. **定义**：按经营与管理需要对产品进行归类的基础对象。
4. **作用**：支持产品组织、筛选、统计和统一口径管理。
5. **生命周期**：建立、启用、被产品引用、调整或停用并保留历史引用。
6. **状态**：启用、停用；不在本任务扩展状态枚举。
7. **关联对象**：产品、SKU、统计报表。
8. **输出**：可供产品归类和统计使用的分类信息。

### BO-B04 品牌

1. **中文名称**：品牌
2. **英文名称**：Brand
3. **定义**：标识产品所属品牌的统一基础对象。
4. **作用**：统一品牌口径，支持产品维护、查询和分析。
5. **生命周期**：建立、启用、被产品引用、调整或停用并保留历史引用。
6. **状态**：启用、停用；不在本任务扩展状态枚举。
7. **关联对象**：产品、SKU、统计报表。
8. **输出**：可供产品引用的标准品牌信息。

### BO-B05 供应商

1. **中文名称**：供应商
2. **英文名称**：Supplier
3. **定义**：向公司提供外购产品或相关供应服务的合作方。
4. **作用**：承载采购来源、供货关系、历史价格和结算关系。
5. **生命周期**：建档、审核启用、发生采购业务、暂停或终止合作并保留历史。
6. **状态**：引用适用的主数据状态；不在本任务新增状态枚举。
7. **关联对象**：采购单、采购付款单、采购退货单、产品、SKU。
8. **输出**：可供采购业务选择和追溯的供应商档案。

### BO-B06 生产厂家

1. **中文名称**：生产厂家
2. **英文名称**：Manufacturer
3. **定义**：接受委外生产安排并完成产品制造的合作方，与供应商独立管理。
4. **作用**：承载生产订单、生产进度、生产付款和厂家暂存责任。
5. **生命周期**：建档、审核启用、发生生产业务、暂停或终止合作并保留历史。
6. **状态**：引用适用的主数据状态；不在本任务新增状态枚举。
7. **关联对象**：委外生产单、生产付款单、厂家仓、质量验收单。
8. **输出**：可供委外生产与厂家库存业务选择和追溯的厂家档案。

### BO-B07 仓库

1. **中文名称**：仓库
2. **英文名称**：Warehouse
3. **定义**：用于区分库存责任地点或业务阶段的库存节点基础对象。
4. **作用**：统一承载公司仓、厂家仓、海外仓、在途和待处理等相互独立的库存节点。
5. **生命周期**：建立、审核启用、承载库存业务、停用并保留历史记录。
6. **状态**：启用、停用；不在本任务扩展状态枚举。
7. **关联对象**：公司仓、厂家仓、海外仓、库存、库存流水、出入库单据。
8. **输出**：可被库存及正式业务单据引用的库存节点信息。

### BO-B08 公司仓

1. **中文名称**：公司仓
2. **英文名称**：Company Warehouse
3. **定义**：公司当前唯一实际仓库对应的库存节点。
4. **作用**：承载合格品入库、国内销售出库、退货、调拨、盘点和调整业务。
5. **生命周期**：建立并启用、持续承载公司实物库存、停用时保留全部历史。
6. **状态**：启用、停用；当前公司实际仓数量为一个。
7. **关联对象**：仓库、库存、入库单、出库单、调拨单、盘点单。
8. **输出**：公司实际仓的独立库存与业务追溯信息。

### BO-B09 厂家仓

1. **中文名称**：厂家仓
2. **英文名称**：Manufacturer Warehouse
3. **定义**：与单一生产厂家对应、用于管理其合格完工品暂存数量的独立库存节点。
4. **作用**：隔离不同厂家暂存库存，支持长期暂存和跨境分批发货。
5. **生命周期**：随厂家建立并启用、接收合格完工品、发生发货或调整、停用时保留历史。
6. **状态**：启用、停用；每个厂家对应独立厂家仓，不得合并库存。
7. **关联对象**：生产厂家、委外生产单、质量验收单、跨境发货单、厂家库存。
8. **输出**：按厂家独立核算的暂存库存及可发数量信息。

### BO-B10 海外仓

1. **中文名称**：海外仓
2. **英文名称**：Overseas Warehouse
3. **定义**：位于海外并通过定期 Excel 导入维护库存的独立库存节点。
4. **作用**：承载跨境目的库存、海外实收和差异追踪。
5. **生命周期**：建立并启用、接收导入确认数据、持续更新库存、停用时保留历史。
6. **状态**：启用、停用；本期不直接对接海外平台或海外仓 API。
7. **关联对象**：跨境发货单、海外库存导入任务、海外库存、电商平台。
8. **输出**：按海外仓独立管理的库存与收货差异信息。

### BO-B11 电商平台

1. **中文名称**：电商平台
2. **英文名称**：E-commerce Platform
3. **定义**：承载国内或跨境电商业务的外部渠道平台对象。
4. **作用**：统一标识订单来源、跨境目的业务和统计维度。
5. **生命周期**：建立、启用、关联店铺和业务、停用并保留历史引用。
6. **状态**：启用、停用；不在本任务扩展状态枚举。
7. **关联对象**：店铺、出库单、跨境发货单、海外仓、统计报表。
8. **输出**：可供店铺和业务单据引用的平台信息。

### BO-B12 店铺

1. **中文名称**：店铺
2. **英文名称**：Store
3. **定义**：公司在特定电商平台下经营的独立店铺对象，与平台分开管理。
4. **作用**：细分销售出库来源、数据权限和经营统计。
5. **生命周期**：在平台下建立、启用、发生业务、停用并保留历史引用。
6. **状态**：启用、停用；不在本任务扩展状态枚举。
7. **关联对象**：电商平台、出库单、销售退货单、统计报表。
8. **输出**：可供销售出库和统计引用的店铺信息。

## 2.6.2 业务对象

本章对象的生命周期统一引用 Task 2.4，状态统一引用 Task 2.5。Task 2.5 未单独定义的对象不在本任务新增状态枚举。所有库存变化必须由正式业务对象触发，已完成业务不得直接删除。

### BO-T01 采购单

1. **中文名称**：采购单
2. **英文名称**：Purchase Order
3. **定义**：记录向供应商采购产品、数量、价格和交付要求的正式业务对象。
4. **作用**：驱动外部采购、到货、验收、入库、退货和付款追踪。
5. **生命周期**：引用 Task 2.4 的国内外部采购生命周期。
6. **状态**：引用 Task 2.5 的采购单状态体系。
7. **关联对象**：供应商、SKU、质量验收单、入库单、采购付款单、采购退货单。
8. **输出**：经审核的采购安排、执行数量与采购完成信息。

### BO-T02 委外生产单

1. **中文名称**：委外生产单
2. **英文名称**：Production Order
3. **定义**：记录向生产厂家下达产品生产数量、价格和交期要求的正式业务对象。
4. **作用**：驱动生产进度、完工、验收、入库或厂家暂存及生产付款追踪。
5. **生命周期**：引用 Task 2.4 的委外生产生命周期。
6. **状态**：引用 Task 2.5 的委外生产单状态体系。
7. **关联对象**：生产厂家、SKU、质量验收单、入库单、厂家仓、生产付款单。
8. **输出**：经审核的生产安排、分批执行数量与生产完成信息。

### BO-T03 质量验收单

1. **中文名称**：质量验收单
2. **英文名称**：Inspection Order
3. **定义**：记录采购到货或生产完工商品质量判定结果的正式业务对象。
4. **作用**：强制区分合格与不合格数量，阻止未验收商品进入正常可用库存。
5. **生命周期**：引用 Task 2.4 的质量验收生命周期。
6. **状态**：引用 Task 2.5 的质量验收单状态体系。
7. **关联对象**：采购单、委外生产单、SKU、入库单、待处理库存。
8. **输出**：合格数量、不合格数量、原因、处理方式和可入库依据。

### BO-T04 入库单

1. **中文名称**：入库单
2. **英文名称**：Inbound Order
3. **定义**：确认合格商品进入指定库存节点的正式库存增加对象。
4. **作用**：依据验收结果增加对应库存并生成库存流水。
5. **生命周期**：引用 Task 2.4 的采购入库、生产入库及退货入库相关生命周期。
6. **状态**：引用 Task 2.5 的入库单状态体系。
7. **关联对象**：质量验收单、采购单、委外生产单、销售退货单、仓库、库存流水。
8. **输出**：经确认的入库结果、库存增加及对应库存流水。

### BO-T05 出库单

1. **中文名称**：出库单
2. **英文名称**：Outbound Order
3. **定义**：记录销售、领用、样品或其他已批准用途导致商品离开库存节点的正式对象。
4. **作用**：校验可用库存、完成库存扣减并保留来源追溯。
5. **生命周期**：引用 Task 2.4 的国内销售出库及其他出库相关生命周期。
6. **状态**：引用 Task 2.5 的出库单状态体系。
7. **关联对象**：SKU、仓库、店铺、电商平台、销售退货单、库存流水。
8. **输出**：经确认的出库结果、库存减少及对应库存流水。

### BO-T06 采购付款单

1. **中文名称**：采购付款单
2. **英文名称**：Purchase Payment
3. **定义**：记录针对采购单发生的付款金额、日期、方式和凭证的正式对象。
4. **作用**：累计采购已付款金额并计算未付款金额和结算状态，不替代财务软件。
5. **生命周期**：引用 Task 2.4 的采购付款生命周期。
6. **状态**：引用 Task 2.5 的采购付款状态体系；付款状态与采购业务完成状态独立。
7. **关联对象**：采购单、供应商、附件、操作日志。
8. **输出**：付款记录、累计付款、未付款金额和结算状态。

### BO-T07 生产付款单

1. **中文名称**：生产付款单
2. **英文名称**：Production Payment
3. **定义**：记录针对委外生产业务发生的付款金额、日期、方式和凭证的正式对象。
4. **作用**：跟踪生产业务结算，不以付款状态代替生产、验收或入库完成状态。
5. **生命周期**：复用 Task 2.4 付款记录的新增、累计、结算与留痕原则。
6. **状态**：复用 Task 2.5 的付款状态体系；与委外生产单业务状态独立。
7. **关联对象**：委外生产单、生产厂家、附件、操作日志。
8. **输出**：生产付款记录、累计付款、未付款金额和结算状态。

### BO-T08 采购退货单

1. **中文名称**：采购退货单
2. **英文名称**：Purchase Return
3. **定义**：记录已入库采购商品因质量或其他确认原因退回供应商的正式对象。
4. **作用**：关联原采购与入库，审核后扣减库存并追踪补货或退款结果。
5. **生命周期**：引用 Task 2.4 的采购退货生命周期。
6. **状态**：引用 Task 2.5 的采购退货单状态体系。
7. **关联对象**：采购单、入库单、供应商、SKU、库存流水。
8. **输出**：退货出库结果、库存变化以及补货或退款处理信息。

### BO-T09 销售退货单

1. **中文名称**：销售退货单
2. **英文名称**：Sales Return
3. **定义**：记录客户或平台退回商品并关联原销售出库的正式对象。
4. **作用**：检查退货商品状态，将可售品和不可售品分别进入适用库存。
5. **生命周期**：引用 Task 2.4 的销售退货生命周期。
6. **状态**：引用 Task 2.5 的销售退货单状态体系。
7. **关联对象**：出库单、店铺、电商平台、入库单、可用库存、待处理库存。
8. **输出**：退货判定、退货入库结果和库存流水。

### BO-T10 跨境发货单

1. **中文名称**：跨境发货单
2. **英文名称**：Cross-border Shipment
3. **定义**：记录从厂家仓或适用来源库存分批发往海外仓的正式业务对象。
4. **作用**：跟踪发货批次、物流、在途、海外实收及数量差异。
5. **生命周期**：引用 Task 2.4 的跨境分批发货生命周期。
6. **状态**：引用 Task 2.5 的跨境发货单状态体系。
7. **关联对象**：委外生产单、厂家仓、海外仓、SKU、在途库存、海外库存导入任务。
8. **输出**：发货批次、在途变化、物流跟踪和海外收货差异信息。

### BO-T11 调拨单

1. **中文名称**：调拨单
2. **英文名称**：Transfer Order
3. **定义**：记录 SKU 在不同库存节点之间转移的正式业务对象。
4. **作用**：分离调出与调入确认，并在两者之间形成在途库存。
5. **生命周期**：引用 Task 2.4 的库存调拨生命周期。
6. **状态**：引用 Task 2.5 的调拨单状态体系。
7. **关联对象**：调出仓、调入仓、SKU、在途库存、库存流水。
8. **输出**：调出、在途、调入结果及各阶段库存流水。

### BO-T12 盘点单

1. **中文名称**：盘点单
2. **英文名称**：Stock Count
3. **定义**：记录指定库存节点账面数量、实物数量和差异确认过程的正式对象。
4. **作用**：通过审核差异生成库存调整依据，不直接覆盖库存余额。
5. **生命周期**：引用 Task 2.4 的库存盘点生命周期。
6. **状态**：引用 Task 2.5 的盘点单状态体系。
7. **关联对象**：仓库、SKU、库存、库存调整单、操作日志。
8. **输出**：盘点结果、差异明细和经审核的调整依据。

### BO-T13 库存调整单

1. **中文名称**：库存调整单
2. **英文名称**：Inventory Adjustment
3. **定义**：依据已确认盘点差异或正式纠错原因调整库存的受控业务对象。
4. **作用**：替代直接修改库存余额，形成可审核、可追溯的库存纠正。
5. **生命周期**：引用 Task 2.4 的库存调整相关生命周期。
6. **状态**：引用 Task 2.5 的库存调整单状态体系。
7. **关联对象**：盘点单、仓库、SKU、库存、库存流水、审批记录。
8. **输出**：经批准的库存调整结果和库存流水。

### BO-T14 报损单

1. **中文名称**：报损单
2. **英文名称**：Damage Report
3. **定义**：记录商品因损坏等原因经批准退出正常库存的正式对象。
4. **作用**：控制报损审批、禁止超库存报损并保留损失原因与库存变化。
5. **生命周期**：引用 Task 2.4 的其他出库与库存异常处理原则。
6. **状态**：引用 Task 2.5 适用的出库类状态体系；不在本任务新增状态枚举。
7. **关联对象**：SKU、仓库、库存、库存流水、操作日志。
8. **输出**：经批准的报损结果、库存减少和原因追溯信息。

### BO-T15 海外库存导入任务

1. **中文名称**：海外库存导入任务
2. **英文名称**：Overseas Inventory Import
3. **定义**：承载海外仓 Excel 文件校验、预览、确认、执行和结果留痕的正式业务对象。
4. **作用**：防止重复导入，显式处理 SKU 匹配、成功失败明细和库存差异。
5. **生命周期**：引用 Task 2.4 的海外仓库存导入生命周期。
6. **状态**：引用 Task 2.5 的海外库存导入任务状态体系。
7. **关联对象**：海外仓、SKU、海外库存、跨境发货单、导入任务、操作日志。
8. **输出**：导入批次、校验结果、成功失败明细及海外库存变化。

### BO-T16 库存预警

1. **中文名称**：库存预警
2. **英文名称**：Inventory Alert
3. **定义**：当产品或 SKU 库存低于已设置安全库存时形成的业务提醒对象。
4. **作用**：支持补货识别、预警查看和后续业务决策，不直接改变库存。
5. **生命周期**：条件触发、生成预警、查看处理、条件解除或关闭。
6. **状态**：引用 Task 2.5 适用的预警状态；未单独定义者不在本任务新增枚举。
7. **关联对象**：SKU、库存、安全库存规则、用户、统计报表。
8. **输出**：低库存提示、补货参考和处理留痕。

### BO-T17 统计报表

1. **中文名称**：统计报表
2. **英文名称**：Report
3. **定义**：基于正式业务数据按确定口径形成的查询、统计或经营展示输出对象。
4. **作用**：提供采购、生产、库存、跨境、付款和预警等管理视图。
5. **生命周期**：按查询条件生成、展示或导出；不形成独立业务事实来源。
6. **状态**：报表属于输出对象，不设置业务完成状态。
7. **关联对象**：各类基础对象、业务对象、库存对象及数据权限。
8. **输出**：经营看板、统计明细、汇总结果和授权范围内的报表文件。

### BO-T18 操作日志

1. **中文名称**：操作日志
2. **英文名称**：Operation Log
3. **定义**：面向业务追溯展示关键操作事件的只追加记录对象，来源于统一审计记录。
4. **作用**：说明谁在何时对何业务对象执行了何种关键操作，不形成平行审计数据源。
5. **生命周期**：随关键操作生成并永久保留，只追加，不允许修改或删除。
6. **状态**：生成后即为有效历史记录，不设置可修改业务状态。
7. **关联对象**：用户、各类正式业务对象、审计日志。
8. **输出**：按权限可查询的业务操作轨迹和追溯记录。

## 2.6.3 库存对象

库存对象遵守单据驱动、流水留痕和禁止直接修改原则。一个 SKU 在一个库存节点只有一条当前库存记录；厂家、在途、海外和待处理库存均按独立库存节点管理。

### BO-I01 库存

1. **中文名称**：库存
2. **英文名称**：Inventory
3. **定义**：某一 SKU 在某一库存节点的当前数量状态。
4. **作用**：作为系统正式库存余额的唯一真实来源，支持可用性校验和库存查询。
5. **生命周期**：由正式业务单据引起增加或减少并持续更新，历史变化由库存流水保留。
6. **状态**：以当前数量状态体现；禁止普通用户直接修改库存数量。
7. **关联对象**：SKU、仓库、库存流水、各类库存变动单据。
8. **输出**：每个 SKU 在每个库存节点唯一的当前库存记录。

### BO-I02 库存流水

1. **中文名称**：库存流水
2. **英文名称**：Inventory Transaction
3. **定义**：每次正式库存变化产生的不可覆盖历史记录。
4. **作用**：记录变动前、增加、减少、变动后数量及业务来源，支持完整追溯。
5. **生命周期**：随库存变动生成并永久保留，不得修改或删除。
6. **状态**：生成后即为有效历史事实，不设置可编辑状态。
7. **关联对象**：库存、SKU、仓库、来源业务对象、操作日志。
8. **输出**：连续、可追溯且不可篡改的库存变化轨迹。

### BO-I03 可用库存

1. **中文名称**：可用库存
2. **英文名称**：Available Inventory
3. **定义**：在指定库存节点中已验收合格且可用于正常销售或业务执行的库存数量。
4. **作用**：作为出库、调拨和报损等操作的数量校验依据，确保不出现负库存。
5. **生命周期**：随合格入库增加，随正式出库或转移减少，由业务单据更新。
6. **状态**：按当前可用数量体现，不单独设置可编辑状态。
7. **关联对象**：库存、SKU、公司仓、入库单、出库单、调拨单。
8. **输出**：指定 SKU 在指定节点可被正常业务使用的数量。

### BO-I04 在途库存

1. **中文名称**：在途库存
2. **英文名称**：In-Transit Inventory
3. **定义**：商品已从来源节点调出或发货、尚未在目的节点确认接收期间的独立库存节点数量。
4. **作用**：分离两端库存变化，反映调拨和跨境运输中的真实业务阶段。
5. **生命周期**：调出或发货确认后增加，调入或海外接收确认后减少，异常时按正式流程处理。
6. **状态**：引用 Task 2.5 的在途状态体系；数量由正式单据更新。
7. **关联对象**：调拨单、跨境发货单、来源仓、目的仓、库存流水。
8. **输出**：按业务批次可追溯的在途数量及运输阶段信息。

### BO-I05 厂家库存

1. **中文名称**：厂家库存
2. **英文名称**：Manufacturer Inventory
3. **定义**：合格完工品存放在对应厂家独立厂家仓中的当前库存数量。
4. **作用**：按厂家隔离暂存责任，并支持分批发货、剩余可发数量和长期暂存查询。
5. **生命周期**：合格完工进入厂家仓后增加，发货、退回或正式调整后减少。
6. **状态**：以厂家仓当前数量体现，禁止跨厂家合并或直接修改。
7. **关联对象**：生产厂家、厂家仓、委外生产单、质量验收单、跨境发货单。
8. **输出**：每个厂家仓内各 SKU 的当前暂存数量和可发信息。

### BO-I06 海外库存

1. **中文名称**：海外库存
2. **英文名称**：Overseas Inventory
3. **定义**：通过经确认的海外库存 Excel 导入维护的海外仓当前库存数量。
4. **作用**：统一管理不同海外仓库存并关联跨境发货、实收和差异。
5. **生命周期**：随已确认导入任务更新，保留导入批次及变化追溯。
6. **状态**：以各海外仓当前数量体现，不允许未匹配 SKU 静默写入。
7. **关联对象**：海外仓、SKU、海外库存导入任务、跨境发货单、库存流水。
8. **输出**：按海外仓和 SKU 独立管理的当前海外库存及变化信息。

### BO-I07 待处理库存

1. **中文名称**：待处理库存
2. **英文名称**：Pending Inventory
3. **定义**：不合格、退货待判定或其他暂不能进入正常可用库存商品的独立库存节点数量。
4. **作用**：隔离不可售或未完成处理的商品，防止其参与正常销售。
5. **生命周期**：经验收或退货判定进入，按退回、报损、返工、转为可售等正式处理流程减少。
6. **状态**：以待处理数量及关联处理过程体现，不得直接转为可用库存。
7. **关联对象**：质量验收单、销售退货单、采购退货单、报损单、库存调整单。
8. **输出**：需后续处理的独立库存数量、原因和处理追溯信息。

## 2.6.4 系统对象

### BO-S01 用户

1. **中文名称**：用户
2. **英文名称**：User
3. **定义**：获得系统账号并代表内部员工执行或查询业务的人员对象。
4. **作用**：承载身份识别、角色分配、数据范围和操作追溯。
5. **生命周期**：创建、分配角色、启用、调整权限、停用并保留历史操作。
6. **状态**：启用、停用；不在本任务扩展状态枚举。
7. **关联对象**：角色、权限、操作日志、审计日志、各类业务对象。
8. **输出**：受控的系统访问身份和可追溯操作主体。

### BO-S02 角色

1. **中文名称**：角色
2. **英文名称**：Role
3. **定义**：按岗位职责组织一组权限的系统管理对象。
4. **作用**：支持功能、操作和数据权限分配以及关键职责分离。
5. **生命周期**：建立、配置权限、分配用户、调整或停用并保留历史关系。
6. **状态**：启用、停用；不在本任务扩展状态枚举。
7. **关联对象**：用户、权限、仓库、业务线。
8. **输出**：可复用的岗位权限集合。建议角色包括系统管理员、仓库管理员、采购员、生产管理员、财务人员和普通查询用户。

### BO-S03 权限

1. **中文名称**：权限
2. **英文名称**：Permission
3. **定义**：控制用户可访问功能、可执行操作和可查看数据范围的授权对象。
4. **作用**：限制审核、反审核、作废、库存调整、付款修改、导入和权限变更等关键操作。
5. **生命周期**：定义、分配至角色、生效、调整或停用并保留变更记录。
6. **状态**：启用、停用；具体授权随角色与用户关系生效。
7. **关联对象**：角色、用户、仓库、业务对象、审计日志。
8. **输出**：用户执行系统操作时的授权判断和数据可见范围。

### BO-S04 系统参数

1. **中文名称**：系统参数
2. **英文名称**：System Setting
3. **定义**：经授权维护、用于控制系统通用业务行为的配置对象。
4. **作用**：集中管理已获确认的业务配置，避免通过临时操作绕过正式规则。
5. **生命周期**：提出配置、授权维护、生效、变更留痕、停用或替换。
6. **状态**：启用、停用；不得以参数修改 Frozen 业务规则。
7. **关联对象**：用户、权限、库存预警、导入任务、审计日志。
8. **输出**：在正式业务规则范围内生效的统一系统配置。

### BO-S05 导入任务

1. **中文名称**：导入任务
2. **英文名称**：Import Task
3. **定义**：统一承载文件接收、格式校验、业务匹配、人工确认、执行和结果记录的系统任务对象。
4. **作用**：为经批准的 Excel 导入提供批次追踪、重复识别和错误反馈能力。
5. **生命周期**：创建、校验、预览、确认、执行、完成或失败并保留记录。
6. **状态**：引用 Task 2.5 适用的导入任务状态体系。
7. **关联对象**：海外库存导入任务、用户、附件、操作日志、审计日志。
8. **输出**：导入批次、处理状态、成功失败明细和错误原因。

### BO-S06 数据备份

1. **中文名称**：数据备份
2. **英文名称**：Backup Task
3. **定义**：记录系统数据和图片备份执行及历史保留情况的系统任务对象。
4. **作用**：支持数据恢复准备和备份执行追溯，不改变正式业务数据。
5. **生命周期**：按计划创建、执行、完成或失败、保留、到期清理并记录结果。
6. **状态**：待执行、执行中、成功、失败、已清理；作为业务级任务状态，不涉及技术方案。
7. **关联对象**：系统参数、用户、操作日志、审计日志。
8. **输出**：备份执行记录和保留结果；数据库每 3 天备份，图片每周增量备份，历史备份 30 天清理。

### BO-S07 操作日志

1. **中文名称**：操作日志
2. **英文名称**：Audit Log
3. **定义**：统一记录用户关键操作、权限变更和系统重要事件的正式审计对象。
4. **作用**：作为系统审计的统一正式来源，支持责任追溯与安全检查。
5. **生命周期**：随受控事件自动生成并永久保留，只追加，不允许修改或删除。
6. **状态**：生成后即为有效审计事实，不设置可编辑状态。
7. **关联对象**：用户、角色、权限、系统参数、业务操作日志及全部受控对象。
8. **输出**：完整、按权限可查询且不可覆盖的系统审计轨迹。

## 3. 跨对象统一规则

1. 产品与 SKU 分开管理，SKU 是最小库存核算单位，本期不管理单把小提琴唯一序列号。
2. 供应商与生产厂家是独立对象，不因同一合作方承担两种职责而混淆业务关系。
3. 公司当前只有一个实际公司仓；每个厂家对应独立厂家仓；海外仓和在途库存均作为独立库存节点管理。
4. 电商平台与店铺分开管理，店铺必须归属相应平台。
5. 付款状态与采购或生产业务完成状态独立，不得互相替代。
6. 所有库存变化必须由正式业务对象触发，已完成业务不得直接删除。
7. 当前库存只能通过正式业务单据更新，禁止直接修改库存数量。
8. 一个 SKU 在一个库存节点只有一条当前库存记录。
9. 库存流水永久保留，不得修改或删除。
10. 待处理库存不得参与正常销售。
11. 统计报表属于输出对象，不形成与正式业务对象平行的数据来源。
12. 操作日志和审计日志只追加，不允许修改或删除；业务操作日志是统一审计记录的业务展示，不形成平行数据源。
