# Task 2.1：系统模块划分（Module Breakdown）

## 1. 任务状态

- 所属阶段：Phase 2 业务流程设计（Business Process Design）
- 任务状态：Completed
- 确认状态：Approved
- 确认人：Project Manager
- 确认日期：2026-07-19

## 2. 任务结论

Violin ERP Lite 第一版固定设置以下九个一级模块：

| 编号 | 一级模块 | 核心职责 |
| --- | --- | --- |
| 01 | 基础资料 | 维护全系统统一主数据 |
| 02 | 采购管理 | 管理采购订单、采购退货和采购付款 |
| 03 | 生产管理 | 管理委外生产、进度和质量验收 |
| 04 | 库存管理 | 展示和管理当前库存状态 |
| 05 | 出入库管理 | 管理所有库存增减和仓库作业 |
| 06 | 跨境业务 | 管理厂家发货、在途和海外库存导入 |
| 07 | 统计分析 | 汇总经营数据、库存数据和预警信息 |
| 08 | 用户权限 | 管理用户、角色、权限和数据范围 |
| 09 | 系统设置 | 管理系统参数、数据工具、日志和初始化 |

## 3. 名称确认

第五个一级模块正式名称保持为“出入库管理”，不得自行改为“库存作业”或其他名称。

## 4. 阶段边界

本任务只确认一级模块数量、名称和核心职责，不包含模块职责详细设计、模块关系设计、数据库设计、页面设计、API 设计、技术选型或业务编码。

## 5. 后续任务

Task 2.1 已完成。后续由 Task 2.2 对九个一级模块的职责、边界、输入、输出及关联进行评审设计。
