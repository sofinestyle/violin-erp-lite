# Phase 1 业务需求分析

## 1. 业务流程

### 1.1 总体业务结构

公司业务分为国内业务和跨境业务。

#### 国内业务

- 面向天猫、抖音等国内电商平台及线下客户；
- 产品来源包括委托生产和外部采购；
- 商品验收后进入公司唯一的实际仓库；
- 国内电商出库按照平台订单逐单登记；
- 系统只管理销售出库，不建设完整销售订单模块。

#### 跨境业务

- 面向 Amazon、Temu 等平台；
- 公司向厂家下达生产或采购订单；
- 商品生产完成后可暂存在厂家仓；
- 每个厂家建立独立厂家仓；
- 商品可一次或分批发往海外；
- 海外仓库存通过 Excel 定期导入；
- 系统管理厂家暂存、发货、在途和海外库存状态。

### 1.2 产品档案流程

提出新增产品需求
→ 收集产品资料
→ 确认分类、型号、规格和单位
→ 沿用或校验现有产品编码/SKU 编码
→ 建立标准产品档案
→ 审核启用
→ 用于采购、验收、库存和出库

规则：

- 同一产品只能有一个标准档案；
- 不同规格应建立不同 SKU；
- 产品产生业务记录后不得物理删除；
- 停用产品保留历史数据；
- 公司已有统一产品编码和 SKU 编码，原则上继续沿用。

### 1.3 国内委外生产流程

提出备货需求
→ 创建委外生产采购单
→ 选择厂家
→ 填写产品、数量、单价和交期
→ 单级审核
→ 厂家生产
→ 跟踪进度
→ 分批或全部完工
→ 质量验收
→ 合格品入公司仓
→ 更新库存和采购付款状态

要求：

- 支持分批完工；
- 支持分批验收；
- 支持分批入库；
- 分别记录订单数量、完工数量、验收数量和入库数量；
- 不合格品不能进入正常可用库存。

### 1.4 国内外部采购流程

提出采购需求
→ 创建采购单
→ 选择供应商
→ 填写产品、数量、单价和交期
→ 单级审核
→ 供应商发货
→ 到货质量验收
→ 合格品入库
→ 更新库存
→ 记录付款情况

支持：

- 分批到货；
- 部分合格；
- 供应商退货；
- 补货；
- 部分付款；
- 多次付款。

### 1.5 国内销售出库流程

取得国内平台订单
→ 创建销售出库单
→ 记录渠道、店铺和平台订单号
→ 选择产品及数量
→ 校验可用库存
→ 单级审核
→ 仓库拣货
→ 确认出库
→ 扣减库存
→ 生成库存流水

要求：

- 国内电商逐单登记；
- 不建设完整销售订单模块；
- 库存不足禁止出库；
- 不允许负库存；
- 平台订单号作为来源依据；
- 已完成出库不得直接删除。

### 1.6 销售退货流程

客户或平台退货
→ 创建退货记录
→ 关联原销售出库
→ 检查商品状态
→ 可售商品返回正常库存
→ 不可售商品进入待处理库存
→ 生成库存流水

### 1.7 跨境采购与厂家暂存流程

根据海外销售计划提出备货需求
→ 创建跨境采购或生产订单
→ 单级审核
→ 厂家生产
→ 更新生产进度
→ 完工质量验收
→ 合格品进入对应厂家暂存仓
→ 形成厂家库存

每个厂家必须建立独立厂家仓。

系统应分别显示：

- 订单数量；
- 未生产数量；
- 生产中数量；
- 已完工数量；
- 厂家暂存数量；
- 已发货数量；
- 剩余可发数量。

### 1.8 跨境分批发货流程

提出海外发货需求
→ 选择来源订单及厂家库存
→ 创建跨境发货单
→ 记录产品、数量、目的仓和物流信息
→ 单级审核
→ 厂家发货
→ 减少厂家库存
→ 增加在途库存
→ 海外库存 Excel 导入
→ 更新海外仓库存
→ 处理数量差异

支持：

- 一张订单分多次发货；
- 发往不同海外仓；
- 不同物流方式；
- 在途库存；
- 海外实收差异；
- 厂家长期暂存。

### 1.9 海外仓库存导入流程

准备海外仓 Excel 文件
→ 上传文件
→ 格式校验
→ 产品/SKU 编码匹配
→ 显示导入预览
→ 人工确认
→ 执行导入
→ 生成导入批次
→ 更新海外库存
→ 保存成功与失败明细

要求：

- 保留原始导入文件或可追溯记录；
- 防止同一文件重复导入；
- 不匹配 SKU 不得静默写入；
- 导入错误必须显示原因；
- 本期不直接对接海外平台 API。

### 1.10 库存调拨流程

创建调拨单
→ 选择调出仓和调入仓
→ 校验库存
→ 单级审核
→ 调出确认
→ 形成在途库存
→ 调入确认
→ 增加调入仓库存

### 1.11 库存盘点流程

创建盘点任务
→ 生成账面库存
→ 实物盘点
→ 录入实际数量
→ 计算差异
→ 审核差异
→ 生成库存调整单
→ 更新库存和流水

不得直接覆盖库存余额。

### 1.12 采购退货流程

发现采购质量问题
→ 创建采购退货单
→ 关联原采购和入库记录
→ 单级审核
→ 出库退回供应商
→ 扣减库存
→ 记录补货或退款结果

### 1.13 采购付款流程

选择采购订单
→ 新增付款记录
→ 填写付款金额、日期和方式
→ 上传或关联付款凭证
→ 更新累计已付款金额
→ 自动计算未付款金额
→ 更新结算状态

结算状态：

- 未付款；
- 部分付款；
- 已付清。

系统只记录付款，不替代财务软件，不生成会计凭证。

## 2. 角色分析

### 2.1 管理员

负责：

- 用户和角色；
- 权限；
- 基础资料；
- 单据审核；
- 库存调整审批；
- 异常处理；
- 系统配置；
- 查看全部业务数据。

### 2.2 采购人员

负责：

- 供应商资料；
- 采购单；
- 委外生产订单；
- 跨境采购；
- 生产进度；
- 采购价格；
- 采购付款登记；
- 厂家暂存库存跟踪；
- 供应商退货处理。

采购人员不得直接修改库存。

### 2.3 仓库人员

负责：

- 到货登记；
- 质量验收；
- 入库；
- 出库；
- 调拨；
- 盘点；
- 退货收发；
- 库存查询。

本期无独立质检人员，质量验收由获得权限的采购人员、仓库人员或管理员执行。

### 2.4 销售人员

负责：

- 国内逐单销售出库申请或登记；
- 记录平台订单号；
- 查看可用库存；
- 退货登记；
- 查询本人或授权渠道数据。

### 2.5 公司负责人

负责：

- 查看经营看板；
- 查看采购和库存；
- 查看厂家暂存；
- 查看在途和海外库存；
- 查看付款情况；
- 查看预警；
- 审批重大异常；
- 查看统计分析。

### 2.6 权限原则

权限分为：

1. 功能权限；
2. 操作权限；
3. 数据权限。

关键操作必须受控：

- 审核；
- 反审核；
- 作废；
- 库存调整；
- 盘点差异确认；
- 付款记录修改；
- 产品停用；
- 用户权限修改；
- Excel 导入。

## 3. 业务对象

以下为本阶段确认的概念级业务对象，不构成数据库、数据表或字段设计。

### 3.1 基础资料

- 产品：公司统一维护的产品主档及其公共信息；
- SKU：区分具体规格并被业务单据引用的库存管理单元；
- 产品分类：用于组织和检索产品的分类资料；
- 品牌：产品所属品牌资料；
- 单位：采购、库存及出库采用的计量单位；
- 供应商：外部采购供货方资料；
- 客户：与国内销售出库及退货追溯相关的客户资料；
- 销售渠道：天猫、抖音、线下等业务来源分类；
- 店铺：销售渠道下的具体经营主体；
- 仓库：公司仓、厂家仓、海外仓及业务所需虚拟仓的统一资料；
- 用户：系统内部使用人员账号；
- 角色：用户职责及权限集合。

### 3.2 采购和生产

- 采购单：记录外部采购业务及其执行状态；
- 委外生产订单：记录委托厂家生产的产品、数量、价格、交期和执行状态；
- 跨境采购订单：记录服务于海外业务的采购或生产安排；
- 采购明细：采购类订单中的产品、数量、单价和执行数量；
- 生产进度：记录未生产、生产中和完工等进度；
- 到货记录：记录采购商品分批到货情况；
- 验收记录：记录质量验收数量、结论和处理方式；
- 采购退货单：记录关联原采购及入库的退供应商业务；
- 采购付款记录：记录采购付款过程和结算状态；
- 供应商价格：记录供应产品的历史采购价格信息。

### 3.3 库存

- 库存余额：各仓库、产品/SKU 当前账面数量；
- 可用库存：通过验收且可被业务占用或出库的库存；
- 在途库存：已调出或已发货、尚未调入或收货的库存；
- 厂家暂存库存：已完工验收并暂存于具体厂家仓的库存；
- 海外库存：由海外仓 Excel 导入并按海外仓管理的库存；
- 待处理库存：不可售或待确定处理结果的隔离库存；
- 库存流水：每次库存变化的完整追溯记录；
- 入库单：触发合格品进入相应仓库的正式单据；
- 出库单：触发销售、领用、报损等库存减少的正式单据；
- 调拨单：管理仓库间调出、在途和调入过程；
- 盘点单：记录账面、实盘和差异；
- 库存调整单：经审核纠正盘盈、盘亏、报损或错误的正式依据；
- 库存批次：用于追溯采购来源、入库时间、数量和去向。

### 3.4 跨境

- 厂家仓：按厂家独立建立的暂存仓；
- 跨境发货单：记录从厂家仓等来源发往海外仓的发货业务；
- 发货批次：一张来源订单被分批发货时的执行批次；
- 物流信息：运输方式、服务商、单号及在途状态；
- 目的海外仓：跨境发货的目标仓库；
- 海外库存导入批次：一次 Excel 导入的文件、结果与追溯信息；
- 海外收货差异：发货数量和海外实际接收数量之间的差异记录。

### 3.5 销售出库

- 销售出库单：按平台订单逐单登记并扣减库存的正式单据；
- 销售退货：关联原销售出库并按商品状态处理退回库存；
- 样品出库：用于样品用途的库存减少单据；
- 活动领用：用于市场或经营活动的库存减少单据；
- 内部领用：公司内部使用商品的库存减少单据；
- 报损出库：经正式流程核准的损耗库存减少单据。

### 3.6 系统管理

- 审批记录：保存单级审核、反审核及相关处理留痕；
- 操作日志：记录用户关键操作及其结果；
- 附件：关联业务单据的付款凭证等文件资料；
- Excel 导入任务：管理导入校验、预览、执行及结果；
- 预警规则：定义安全库存和补货提醒条件；
- 消息通知：向授权用户传递预警和业务处理提醒。

## 4. 核心业务

1. 产品主数据管理：建立唯一、可停用但不可破坏历史的标准产品档案。
2. SKU 管理：按规格维护正式库存管理单元并沿用现有编码。
3. 供应商管理：统一维护供应商、合作状态、供应产品、交期与历史价格。
4. 采购管理：覆盖创建、单级审核、分批到货、验收、入库、退货和付款记录。
5. 委外生产管理：跟踪厂家、数量、价格、交期、分批完工及后续验收入库。
6. 生产进度管理：分别呈现未生产、生产中、已完工等数量和状态。
7. 强制质量验收：任何采购或生产商品进入正常可用库存前必须验收。
8. 公司仓库存管理：管理公司唯一实际仓库的收发存、盘点和调整。
9. 厂家独立暂存仓：每个厂家独立管理已完工验收后的暂存库存。
10. 跨境分批发货：支持同一订单多次、向不同海外仓和采用不同物流方式发货。
11. 在途库存：记录调出或跨境发出后、调入或海外收货前的库存状态。
12. 海外库存 Excel 导入：通过可校验、预览、确认和追溯的批次机制更新海外库存。
13. 国内逐单销售出库：按平台订单逐单登记来源、审核、拣货和出库。
14. 采购付款记录：记录多次或部分付款、凭证、累计金额和结算状态。
15. 盘点与库存调整：通过盘点差异和正式调整单更新库存，不直接覆盖余额。
16. 安全库存和补货预警：当产品或 SKU 低于安全库存时产生提醒。
17. 审核和追溯：关键单据单级审核，业务变更、库存变化和操作均留痕。
18. 统计分析：提供采购、库存、厂家暂存、在途、海外、付款和预警分析。
19. 微信小程序：供内部员工处理授权范围内的移动业务。
20. PC 管理端：用于基础资料、批量导入、复杂查询和报表。

微信小程序和 PC 管理端共用同一数据源；微信小程序仅供内部员工使用；PC 端用于基础资料、批量导入、复杂查询和报表；不允许形成小程序和 PC 两套独立数据。

## 7. 现有问题

1. 多个 Excel 分别管理，业务信息分散且维护成本高。
2. 数据源不统一，同一指标可能存在多个口径。
3. 产品名称和编码可能存在历史不一致，影响匹配和统计。
4. 库存依赖人工汇总，难以保证准确性。
5. Excel 更新不及时，无法反映实时库存。
6. 厂家暂存库存不清晰，难以确定各厂家实际可发数量。
7. 分批发货难追踪，来源订单、批次和剩余数量容易脱节。
8. 采购、生产、验收和库存脱节，业务链路无法完整追溯。
9. 缺少标准单据状态，执行进度和责任边界不清。
10. 缺少库存流水，无法可靠解释库存变化。
11. 缺少审批留痕，关键操作难以审计。
12. 退货、报损和盘点处理不规范，可能直接改动结果数据。
13. 权限不足，功能、操作和数据访问缺少细分控制。
14. 采购付款缺少统一跟踪，未付款金额和结算状态不清。
15. 海外仓数据缺少标准导入机制，错误和重复导入风险高。
16. 管理统计滞后，经营决策依赖临时人工整理。
17. 历史 Excel 数据质量不一致，迁移前需要校验和清理。
18. 无法可靠回答商品从哪里来、现在在哪里、为什么变化。

## 8. 优化方案

1. 建立统一产品中心，所有业务引用唯一产品主数据。
2. 沿用并校验现有产品/SKU 编码，避免平行编码体系。
3. 建立单据驱动的库存模式，库存变化必须有正式业务依据。
4. 所有库存变化形成流水，实现变动前后数量和来源追溯。
5. 国内与跨境共用基础数据，统一产品、供应商、仓库和人员口径。
6. 每个厂家建立独立厂家仓，分别呈现暂存和可发库存。
7. 建立在途库存，明确调出或发货到收货之间的状态。
8. 建立标准海外库存 Excel 导入机制，支持校验、预览、确认、批次和错误明细。
9. 建立分批完工、验收、入库和发货机制，分别维护各执行数量。
10. 建立强制质量验收，不合格品不得进入可用库存。
11. 建立采购付款记录，跟踪部分付款、多次付款和结算状态。
12. 建立单级审核，对关键业务单据和调整操作留痕。
13. 禁止负库存，库存不足时阻止出库、调拨或报损完成。
14. 建立安全库存和补货预警，提高备货及时性。
15. 套装作为独立 SKU 整体管理，销售时扣减套装库存。
16. 建立 PC 端和小程序协同，两端共用后端、数据和业务规则。
17. 建立 Excel 期初迁移方案，只迁移必要主数据、期初库存及必要未完成业务数据。
18. 建立数据质量校验，识别编码、名称、数量和导入格式问题。
19. 建立经营看板，汇总采购、库存、厂家暂存、在途、海外、付款和预警。
20. 分阶段建设，避免过度复杂，未确认不得进入下一阶段。

## 9. 最终需求确认

以下业务结论已经确认：

1. 公司目前只有一个实际公司仓库；
2. 每个生产厂家建立独立厂家仓；
3. 海外仓库存通过 Excel 定期导入；
4. 国内电商按订单逐单登记销售出库；
5. 本期只管理销售出库，不建设完整销售订单；
6. 公司已有统一产品编码和 SKU 编码；
7. 不管理单把小提琴唯一序列号；
8. 套装作为独立 SKU 整体管理；
9. 采购订单统一使用人民币；
10. 系统需要记录采购付款情况；
11. 所有商品入库前必须质量验收；
12. 本期不设置独立质检人员角色；
13. 采用单级审核；
14. 不允许负库存；
15. 支持安全库存和补货预警；
16. 历史 Excel 只导入期初库存，不导入全部历史流水；
17. 微信小程序仅供内部员工使用；
18. 同时建设 PC 管理端；
19. 本期不管理货架和库位；
20. 本期不记录批次质量照片；
21. 采购、生产、验收、入库和发货均应支持分批执行；
22. 公司仓、厂家仓、在途和海外仓库存分别管理；
23. 本期不进入数据库设计。
